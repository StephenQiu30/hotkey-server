# 项目文档

`docs/` 只保存会长期约束 HotKey Server 需求、架构、数据模型和实现验收的正式文档。

当前权威入口是 [设计索引](design/README.md)，其中 `001`–`012` 覆盖产品需求、单体架构、数据、API、采集、去重、多语言相关性、知识交付、事件聚类、热度趋势、AI运行与River流水线。

## 维护规则

- 架构、模块边界、数据库模型、API 契约或运行方式变化时，同步更新对应设计文档。
- 数据库的唯一可执行事实源是完整 `db/schema.sql`；文档不能把未实现目标描述为已实现能力。
- 临时计划、调试日志、会议流水和一次性验收记录不进入 `docs/`。
- 旧编号文档不作为历史兼容方案保留；替代设计确认后直接从仓库清除。
