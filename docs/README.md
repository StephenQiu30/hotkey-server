# 项目文档

`docs/` 只保存会长期约束 HotKey Server 需求、架构、数据模型和实现验收的正式文档。

当前权威入口是 [设计索引](design/README.md)，其中 `001`–`008` 分别覆盖需求、模块化单体架构、数据库、统一 Result、来源采集、标准化去重、多语言相关性，以及 Obsidian/报告交付。

## 维护规则

- 架构、模块边界、数据库模型、API 契约或运行方式变化时，同步更新对应设计文档。
- 数据库的可执行事实源是 `db/migrations/`；文档不能把未实现目标描述为已实现能力。
- 临时计划、调试日志、会议流水和一次性验收记录不进入 `docs/`。
- 旧编号文档不作为历史兼容方案保留；替代设计确认后直接从仓库清除。
