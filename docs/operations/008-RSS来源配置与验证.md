# RSS 来源配置与验证

本文说明如何将公开的 RSS、RSS 1.0（RDF）或 Atom Feed 登记为 HotKey 来源连接，并通过真实接口执行健康探测。

`docs/` 中的配置是填写参考，不会被服务自动加载。正式运行数据必须通过“来源管理”页面或 `POST /api/v1/source-connections` 写入数据库。

## 1. 当前能力边界

当前解析器支持：

- RSS 2.0
- RSS 1.0 / RDF，包含常见的 `dc:creator`、`dc:date` 与 `content:encoded`
- Atom 1.0
- 公开 HTTPS Feed

当前不应描述为已经支持：

- 需要登录、Cookie 或动态签名的私有 Feed
- 把 Token、API Key 或密码直接写在 URL 中
- 跨域跳转的 Feed；重定向后的主机名必须与登记的主机名一致
- HTTP、非 443 端口、IP 地址或内网地址

`auth_type` 和 `credential_ref` 已存在于控制面契约中，但当前 RSS 连接器没有将凭据注入外部请求。公开 Feed 应使用 `auth_type: "none"`；私有 Feed 在连接器完成凭据注入和验收前不得标记为可用。

## 2. 在前端页面中填写

进入“来源管理”→“新增来源”，按下表填写：

| 页面字段 | 填写方式 | 示例 |
|---|---|---|
| 名称 | 使用“机构或期刊 · 栏目”格式，便于检索 | `Nature · Latest Research` |
| 来源类型 | RSS、RDF 和 Atom 都选择 `RSS / Atom` | `RSS / Atom` |
| 接口地址 | 填写官方 HTTPS Feed 地址 | `https://www.nature.com/nature.rss` |

页面创建时默认提交：

```json
{
  "source_type": "rss",
  "auth_type": "none",
  "enabled": true
}
```

页面目前只提供基础字段。归因、保留期、语言、地区、限速和条款地址等高级配置，需要通过 API 创建或更新。

## 3. 推荐的完整 API 配置

创建来源要求管理员 Bearer Token：

```http
POST /api/v1/source-connections
Authorization: Bearer <ADMIN_ACCESS_TOKEN>
Content-Type: application/json
```

公开学术 Feed 示例：

```json
{
  "name": "Nature · Latest Research",
  "source_type": "rss",
  "endpoint": "https://www.nature.com/nature.rss",
  "auth_type": "none",
  "credential_ref": "",
  "enabled": true,
  "terms_policy_url": "https://www.nature.com/info/terms-and-conditions",
  "config": {
    "allow_body_storage": false,
    "requires_attribution": true,
    "requires_deletion_sync": false,
    "content_retention_days": 30,
    "metrics_retention_days": 90,
    "allowed_languages": ["en"],
    "allowed_regions": [],
    "rate_limit_per_minute": 20,
    "request_timeout_seconds": 30,
    "max_pages_per_run": 1
  }
}
```

主流媒体 Feed 示例：

```json
{
  "name": "BBC News · Top Stories",
  "source_type": "rss",
  "endpoint": "https://feeds.bbci.co.uk/news/rss.xml",

  --data '{
    "name": "PLOS ONE · New Articles",

    "endpoint": "https://journals.plos.org/plosone/feed/atom",
    "auth_type": "none",
    "credential_ref": "",

```

## 4. 配置字段说明

| JSON 字段 | 必填 | 约束或建议 |
|---|---:|---|
| `name` | 是 | 1–120 个 Unicode 字符；推荐“来源 · 栏目” |
| `source_type` | 是 | RSS、RDF、Atom 均填写 `rss` |
| `endpoint` | 是 | 仅 HTTPS、443 端口、DNS 主机名；禁止 userinfo、fragment 和凭据型查询参数 |
| `auth_type` | 是 | 公开 Feed 使用 `none` |
| `credential_ref` | 条件 | `none` 时必须为空；其他类型只能使用 `env:NAME`，不能写真实密钥 |
| `enabled` | 否 | 默认建议 `true`；上线前仍需健康探测 |
| `terms_policy_url` | 否 | 推荐填写来源官方 RSS、转载或使用条款页面 |
| `config` | 否 | 省略时使用后端默认值；提交未知字段会被拒绝 |

### `config` 字段

| 字段 | 默认值 | 合法范围 | 使用建议 |
|---|---:|---:|---|
| `allow_body_storage` | `false` | boolean | 未确认转载许可时保持 `false` |
| `requires_attribution` | `false` | boolean | 第三方期刊和媒体通常设置为 `true` |
| `requires_deletion_sync` | `false` | boolean | 来源条款要求同步撤稿或删除时设置为 `true` |
| `content_retention_days` | `30` | 1–3650 | 新闻建议 30；学术摘要可按条款调整 |
| `metrics_retention_days` | `30` | 1–3650 | 推荐 90，便于观察来源质量 |
| `allowed_languages` | `[]` | 最多 8 个 BCP 47 标签 | 例如 `en`、`zh-Hans`；空数组表示不限制 |
| `allowed_regions` | `[]` | 最多 8 个 ISO alpha-2 代码 | 例如 `US`、`GB`、`CN`；空数组表示不限制 |
| `rate_limit_per_minute` | `60` | 1–600 | 外部 Feed 推荐 10–20，避免高频抓取 |
| `request_timeout_seconds` | `30` | 1–120 | 推荐 30 |
| `max_pages_per_run` | `1` | 1–20 | 普通 Feed 推荐 1；只有明确支持同域分页时才增加 |

提交部分 `config` 时，未提供的字段会使用默认值。例如只声明归因要求：

```json
{
  "config": {
    "requires_attribution": true
  }
}
```

## 6. 更新现有来源

先通过管理员身份获取来源详情，读取最新 `version`：

```http
GET /api/v1/source-connections/{id}
Authorization: Bearer <ADMIN_ACCESS_TOKEN>
```

更新时必须提交 `expected_source_version`，避免覆盖并发修改：

```bash
curl --fail-with-body \
  --request PATCH \
  --url "http://127.0.0.1:8080/api/v1/source-connections/5" \
  --header "Authorization: Bearer ${HOTKEY_ACCESS_TOKEN}" \
  --header "Content-Type: application/json" \
  --data '{
    "expected_source_version": 1,
    "config": {
      "allow_body_storage": false,
      "requires_attribution": true,
      "requires_deletion_sync": false,
      "content_retention_days": 30,
      "metrics_retention_days": 90,
      "allowed_languages": ["en"],
      "allowed_regions": [],
      "rate_limit_per_minute": 20,
      "request_timeout_seconds": 30,
      "max_pages_per_run": 1
    }
  }'
```

收到 `409` 时不要重复使用旧版本号，应重新获取详情并确认差异。

## 7. 健康探测

创建或更新后执行：

```bash
curl --fail-with-body \
  --request POST \
  --url "http://127.0.0.1:8080/api/v1/source-connections/5/health" \
  --header "Authorization: Bearer ${HOTKEY_ACCESS_TOKEN}"
```

成功响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "healthy": true,
    "checked_at": "2026-07-18T10:00:00Z"
  }
}
```

常见失败：

| `error_code` | 含义 | 处理方式 |
|---|---|---|
| `destination_not_permitted` | DNS、重定向或目标地址触发 SSRF 防护 | 检查是否解析到内网、保留地址或发生跨域跳转；不得关闭安全校验绕过 |
| `request_failed` | 网络、TLS 或超时错误 | 检查出口网络、证书和超时配置 |
| `upstream_status` | 来源返回非 2xx | 检查 Feed 是否停用、限流或需要授权 |
| `invalid_source_connection` | 来源配置与连接器不一致 | 重新获取来源详情，确认 endpoint、类型和版本 |

在使用流量代理的本地环境中，公共域名可能被解析到 `198.18.0.0/15`。该网段属于基准测试保留地址，后端会正确返回 `destination_not_permitted`。应调整本地 DNS 或代理出口，不得扩大连接器的允许网段。

## 8. 上线检查清单

- Feed URL 来自来源官方页面，并且当前返回有效 XML
- 格式属于 RSS 2.0、RSS 1.0/RDF 或 Atom
- 页面或 API 中没有真实密钥、Token、Cookie
- `terms_policy_url` 指向最新官方条款
- 已根据条款设置归因、正文存储、删除同步和保留期
- 限速和超时不会对来源造成不必要压力
- 健康探测成功，或环境限制已被明确记录
- 管理员确认来源名称、栏目和语言范围
- 采集结果只保留被条款允许的标题、摘要和链接

## 9. 回退

配置错误时优先停用，不要直接删除：

```http
POST /api/v1/source-connections/{id}/disable
Content-Type: application/json

{
  "expected_source_version": 1
}
```

确认不再被监控或采集任务引用后，再使用归档操作。恢复时重新检查官方 Feed 和条款，不能沿用历史健康状态作为当前可用证据。
