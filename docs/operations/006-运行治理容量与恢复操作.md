# 运行治理、容量与恢复

## 运行总览

管理员使用 `GET /api/v1/operations/overview` 查看 River 队列的 available、running、completed、discarded、cancelled 数量和最旧等待时间。接口只返回安全统计，不返回任务参数、Token、模型响应或供应商错误。

## 可重复验证

```bash
HOTKEY_TEST_DSN='postgres:///hotkey_server_dev?sslmode=disable' \
HOTKEY_TEST_REDIS_URL='redis://127.0.0.1:6379/15' make ci
HOTKEY_TEST_DSN='postgres:///hotkey_server_dev?sslmode=disable' \
  HOTKEY_CAPACITY_ROWS=1000 sh test/tools/generate-capacity-fixture.sh
make smoke
```

容量脚本只写入可丢弃 fixture，使用游标分页并验证 `contents_source_published_idx`；不得在生产库直接运行。临时数据库、fixture 和构建产物必须在命令结束后清理。

## 备份与恢复

维护窗口先停止 API/worker，再使用目标数据库的 `pg_dump --format=custom` 和 `pg_restore --list` 验证备份。恢复到隔离库后执行 `db verify`、Schema 校验、OpenAPI/架构校验和关键集成测试；任一失败都保持服务停止并回滚到已验证备份。运行手册不记录真实 DSN、密钥或个人路径。

## 故障边界

- PostgreSQL、Vault、MinIO 或 SMTP 不可用时，任务进入可观测的 retry/discarded 状态，不泄露依赖原始错误。
- 对账只报告缺失、孤儿和哈希冲突，不覆盖人工编辑；修复必须通过新的提案或前向修复。
- 清理只接受白名单 data class，业务事实软删除，追加日志和尝试记录按策略物理删除。
