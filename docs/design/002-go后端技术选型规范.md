# Go 后端技术选型规范

## 1. 文档定位

### 1.1 与 001 的关系

| 文档 | 层级 | 回答的问题 |
|------|------|------------|
| [`001-x热点监控平台设计.md`](001-x热点监控平台设计.md) | 业务架构 | 做什么、模块边界、数据流 |
| **本文档 002** | 工程架构 | 用什么库、为什么、如何集成 |
| [`009-go框架工程化计划.md`](../plans/009-go框架工程化计划.md) | 实施 | 按层迁移的任务与验收 |

业务设计变更走 001；**任何基础设施选型变更必须先更新本文档 ADR**。

### 1.2 主路径决策（ADR-001）

**Chosen：组合型全栈规范**

| 组件 | 版本策略 |
|------|----------|
| uber-go/fx | 最新稳定 minor |
| google/wire | 最新稳定 |
| danielgtaylor/huma/v2 | 最新稳定 v2 |
| sqlc | 最新稳定 |
| hibiken/asynq | 最新稳定 |
| redis/go-redis/v9 | 最新稳定 v9 |
| spf13/cobra, spf13/viper | 最新稳定 |
| go-playground/validator/v10 | 最新稳定 |

**Rejected（一体化框架主路径）：**

| 方案 | 拒绝理由 |
|------|----------|
| Kratos | 骨架迁移面大；对外 REST/OpenAPI 需额外设计；与现有 domain 包冲突 |
| go-zero | 强依赖 goctl 生成；与手写 Repository/Service 风格冲突 |
| Go-kit | 非开箱即用；不减少规范文档工作量 |
| Gin/Chi 作为主 HTTP 框架 | 不原生产出 OpenAPI；与 hotkey-web/miniapp 契约源约束不匹配 |

## 2. 现状与目标差异

| 项 | 001 / 旧 002 | 当前代码 | 目标（本文档） |
|----|--------------|----------|----------------|
| 入口 | `cmd/api` + `cmd/worker` | 单二进制 `cmd/api api\|worker` | `cmd/hotkey` + Cobra |
| PG 访问 | `internal/store/postgres` | `internal/database` + raw SQL | **保留 `internal/database`** + **sqlc** |
| Redis | `internal/store/redis` | 仅 `REDIS_ADDR` 配置，未使用 | `internal/platform/redis` + go-redis |
| 队列 | Redis 辅助 | `jobs.Runner` ticker | **asynq** + 现有 jobs 编排 |
| HTTP | 未定义 | stdlib ServeMux + 手写路径 | **Huma v2** |
| OpenAPI | 契约源 | 未落地 `docs/openapi.json` | Huma export |
| DI | 未定义 | main.go 手工装配 | **Wire + Fx** |

## 3. 架构总览（五平面）

```
┌─────────────────────────────────────────────────────────┐
│ Contract Plane: docs/openapi.json (Huma export)         │
├─────────────────────────────────────────────────────────┤
│ API Plane: Huma → Middleware → Domain Handlers          │
├─────────────────────────────────────────────────────────┤
│ Worker Plane: asynq Server → internal/jobs              │
├─────────────────────────────────────────────────────────┤
│ Store Plane: sqlc → PostgreSQL ; go-redis → Redis       │
├─────────────────────────────────────────────────────────┤
│ Platform Plane: Fx + Wire + Viper + observability       │
└─────────────────────────────────────────────────────────┘
```

数据流（与 001 一致，工程层显式化）：

```
HTTP → Huma Handler → Service → Repository(sqlc) → PostgreSQL
asynq Task → jobs.* → PlatformConnector(x) / Scorer / Notifier
```

## 4. 分层选型（Chosen / Alternatives / Rejected / Integration）

### L1 语言与模块（ADR-002）

| 项 | 内容 |
|----|------|
| **Chosen** | Go 1.26+；单 module `github.com/StephenQiu30/hotkey-server` |
| **Alternatives** | 多 module monorepo |
| **Rejected** | 多 module — MVP 无跨服务复用需求 |
| **Integration** | `go.mod` pin；CI `go test ./...` |

### L2 CLI（ADR-003）

| 项 | 内容 |
|----|------|
| **Chosen** | spf13/cobra：`hotkey api`、`hotkey worker` |
| **Alternatives** | urfave/cli；`os.Args` 手写（现状） |
| **Rejected** | 手写 Args — 子命令增多后不可维护 |
| **Integration** | `cmd/hotkey/main.go` 仅注册命令；Dockerfile `CMD` 指向 `./hotkey-server api` |

### L3 配置（ADR-004）

| 项 | 内容 |
|----|------|
| **Chosen** | spf13/viper：`.env` + 环境变量 + 默认值 |
| **Alternatives** | koanf；裸 `os.Getenv`（现状） |
| **Rejected** | 裸 Getenv — 无统一校验与文档化字段表 |
| **Integration** | 必填：`DATABASE_URL`、`JWT_SECRET`；可选：`REDIS_ADDR`；compose 覆盖容器内 `DATABASE_URL` |

### L4 应用容器（ADR-005）

| 项 | 内容 |
|----|------|
| **Chosen** | uber-go/fx：生命周期、优雅启停、`fx.Module` 按域注册 |
| **Alternatives** | 纯 Wire 手动 main；dig |
| **Rejected** | 纯手写 main — 已证明膨胀至 ~300 行 |
| **Integration** | `internal/app/api.go` / `worker.go` 调用 `fx.New(...).Run()` |

### L5 依赖注入图（ADR-006）

| 项 | 内容 |
|----|------|
| **Chosen** | google/wire 生成 `InitializeAPI` / `InitializeWorker` |
| **Alternatives** | 仅 Fx Provide；手动 new |
| **Rejected** | Fx 单独承担全部绑定 — 编译期无法发现缺失依赖 |
| **Integration** | **Wire 负责「谁注入谁」；Fx 负责「何时启动/停止」**；`//go:generate wire` 在 `internal/app/` |

### L6 HTTP + OpenAPI（ADR-007）★

| 项 | 内容 |
|----|------|
| **Chosen** | **danielgtaylor/huma/v2** — 路由、请求/响应 schema、OpenAPI 一体 |
| **Alternatives** | Gin + swag；Chi + oapi-codegen；ogen |
| **Rejected** | Chi/Gin 主路由 — 契约需二次维护，易与实现漂移 |
| **Rejected** | Phase 先 Chi 后 Huma — 双迁返工 |
| **Integration** | `internal/platform/http/` 注册 API；`make openapi` → `docs/openapi.json`；hotkey-web 用 `@umijs/openapi` 生成客户端 |

**OpenAPI 约定：**

- 统一错误体：`{ "error": string, "code"?: string }`
- 鉴权：Bearer JWT，security scheme 写入 OpenAPI
- 版本前缀：`/api/v1`

### L7 中间件（ADR-008）

| 项 | 内容 |
|----|------|
| **Chosen** | Huma middleware 链：JWT Auth、RequestID、Recover、Metrics |
| **Alternatives** | 独立 Chi middleware 链 |
| **Rejected** | 双 middleware 体系 |
| **Integration** | JWT 解析后写入 context userID；smoke 模式通过 config profile 绕过 |

### L8 校验（ADR-009）

| 项 | 内容 |
|----|------|
| **Chosen** | go-playground/validator/v10 + Huma 输入绑定 |
| **Alternatives** | 手写 if 校验（现状部分） |
| **Rejected** | 纯手写 — 重复且 OpenAPI 难同步 |
| **Integration** | DTO struct tag `validate:"required,email"` |

### L9 持久化（ADR-010）★

| 项 | 内容 |
|----|------|
| **Chosen** | **sqlc** + `database/sql` + **pgx/v5** driver；Repository 端口不变 |
| **Alternatives** | 手写 SQL（现状）；GORM；Ent |
| **Rejected** | GORM — 与 Repository 测试风格、SQL 可控性冲突 |
| **Rejected** | sqlc「可选」— 持久化层必须定论 |
| **Integration** | `db/queries/*.sql` + `sqlc.yaml` → `internal/database/sqlc/`；domain `Repository` 调用 sqlc 生成方法 |

### L10 缓存（ADR-011）

| 项 | 内容 |
|----|------|
| **Chosen** | redis/go-redis/v9 |
| **Alternatives** | 内存 cache；无缓存 |
| **Rejected** | 忽略 Redis — 与 001 设计矛盾 |
| **Integration** | 用途：采集去重键、速率限制计数、短 TTL 缓存；`internal/platform/redis/` |

### L11 任务队列（ADR-012）★

| 项 | 内容 |
|----|------|
| **Chosen** | hibiken/asynq（Redis 后端） |
| **Alternatives** | machinery；纯 `jobs.Runner` ticker（现状）；Kafka |
| **Rejected** | Kafka — 001 非目标 |
| **Rejected** | 永久 ticker-only — 无法重试、死信、可观测 |
| **Integration** | API 可 enqueue；Worker 进程跑 asynq server + 现有 `internal/jobs` 逻辑；与 Fx worker module 绑定 |

**边界：** 轻量定时（health metric）可保留 ticker；**业务 job（poll/aggregate/dispatch）走 asynq**。

### L12 外部平台（ADR-013）

| 项 | 内容 |
|----|------|
| **Chosen** | 保持 `internal/platform/x`；`PlatformConnector` 端口 |
| **Alternatives** | 业务层直接调 X SDK |
| **Rejected** | 直连 — 违反 001 适配器边界 |
| **Integration** | Wire 注入 `x.NewClient(token, baseURL)` |

### L13 观测（ADR-014）

| 项 | 内容 |
|----|------|
| **Chosen** | 标准库 `log/slog` + 现有 `internal/observability` |
| **Alternatives** | uber-go/zap 全面替换 |
| **Rejected** | 现阶段全面换 zap — 非必须 |
| **Integration** | 保留 `RenderLog` 直至 Layer 2；逐步迁移 slog 字段规范 |

### L14 测试（ADR-015）

| 项 | 内容 |
|----|------|
| **Chosen** | stdlib `testing` + `httptest`；**`tests/` 目录独立**（见 ADR-017）；`scripts/smoke-api.sh`；`make validate` 门禁 |
| **Alternatives** | testcontainers-go（PG 集成测试）；测试与源码同包 colocate |
| **Rejected** | 无 smoke 门禁 — 已证明 runtime 接线遗漏；`internal/` 内保留 `*_test.go` — 与目录干净目标冲突 |
| **Integration** | 新 HTTP 层必须更新 `tests/unit/server/` router/wiring 测试 |

### L14b 测试目录（ADR-017）

| 项 | 内容 |
|----|------|
| **Chosen** | 分层 `tests/`：`unit/` + `integration/` + `testutil/` + `fixtures/`；外部测试包 `xxx_test` |
| **Alternatives** | 扁平 `tests/*.go`；`internal/testsupport/` 放 fakes |
| **Rejected** | 全部测试继续 colocate — 耦合高、目录不清晰 |
| **Integration** | 详见 [`003-test模块独立设计.md`](003-test模块独立设计.md)；实施见 [`010-test模块独立计划.md`](../plans/010-test模块独立计划.md) |

### L15 构建与部署（ADR-016）

| 项 | 内容 |
|----|------|
| **Chosen** | 现有 Dockerfile + docker-compose + Makefile |
| **Alternatives** | Helm/K8s 模板 |
| **Rejected** | 首版 K8s 复杂编排 — 001 非目标 |
| **Integration** | 入口改为 `./hotkey-server` 二进制；compose 注入 JWT/DB/Redis |

- 001 中 `internal/store/redis` → 文档修订为 **`internal/platform/redis`**

## 6. 迁移路线（按层）

| Layer | 范围 | 验收 |
|-------|------|------|
| **L0** | 本文档 + 009 计划冻结 | 002/009/001 一致 |
| **L1** | Cobra + Viper + Fx + Wire | `make validate` 全绿；main < 50 行 |
| **L2** | Huma + OpenAPI + middleware + validator **一次完成** | `docs/openapi.json` 存在；smoke 通过 |
| **L3** | sqlc + go-redis + asynq 基础设施 | 集成测试可连 PG/Redis |
| **L4** | 业务 Repository / Worker 闭环 | jobs 链路无生产 stub |

**L4 与 L1-L3 分离：** 业务 stub 补齐不属于「框架选型」，走独立业务 Epic。

## 8. 非目标

1. 不引入 GORM/Ent 作为主持久化路径
2. 不整体迁移 Kratos/go-zero
3. 不拆微服务、不上 gRPC（除非新 ADR）
4. 不在 L1-L2 修改业务规则（仅搬迁 HTTP/DI 基础设施）
5. 不破坏 `scripts/validate-repository.sh`
