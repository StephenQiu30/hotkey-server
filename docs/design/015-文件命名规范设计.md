# 文件名命名规范设计

> **日期：** 2026-07-09
> **状态：** 已批准

## 目标

统一 `controller/`、`service/`、`repository/` 三层文件的命名规则，解决同名文件跨层检索困难的问题，降低理解和调用成本。

## 问题

当前三层中有 7 组同名文件分布在 `controller/`、`service/`、`repository/` 三个包：

| 文件名 | controller | service | repository |
|--------|-----------|---------|------------|
| auth.go | ✅ | ✅ | — |
| monitor.go | ✅ | ✅ | — |
| report.go | ✅ | ✅ | ✅ |
| topic.go | ✅ | ✅ | — |
| trend.go | ✅ | ✅ | ✅ |
| notify.go | ✅ | ✅ | — |
| content.go | ✅ | — | ✅ |

跨包搜索时编辑器返回多个同名结果，需要手动区分层属。此外 repository 层当前混用 `_repo` 和后缀 (`monitor_repo.go` vs `content.go`)，规则不统一。

## 规则

**三层统一加层后缀，一条规则覆盖所有：**

| 包 | 后缀 | 示例 |
|------|--------|-------|
| `controller/` | `_controller` | `auth_controller.go` |
| `service/` | `_service` | `auth_service.go` |
| `repository/` | `_repository` | `monitor_repository.go` |

### 不修改的包

以下包的包名或文件名已有天然区分度：

- `model/entity/`、`model/dto/`、`model/enum/`、`model/vo/` — 在 `model/` 子包下，包名本身是区分
- `queue/`、`worker/`、`config/`、`constant/`、`pkg/`、`module/`、`platform/` — 文件名独一无二或无跨层冲突
- `doc.go` — 包文档文件，不携带类型
