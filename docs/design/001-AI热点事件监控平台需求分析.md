# AI 热点事件监控平台需求分析

本文定义平台首个可交付版本。平台服务内容创作者和新媒体运营，通过用户确认的热点词，在合规的中英文公开来源中发现相关内容，并将多来源报道聚合成可追溯的热点事件卡片。

## 背景

创作者需要在大量新闻、帖子和视频中持续发现与选题相关的热点。单纯按关键词罗列内容会产生重复报道、跨语言割裂和不同平台热度不可比等问题。

平台需要把原始内容转换为事件级结果，同时保留来源证据。AI 用于扩展检索意图、辅助跨语言理解和生成摘要，不能替代来源事实。

## 目标用户

首版服务：

- 内容创作者
- 新媒体编辑
- 小型内容运营团队

首版不以企业舆情、证券研究或大众资讯门户为主要场景。

## 核心用户任务

用户需要完成以下闭环：

1. 创建监控任务并填写核心热点词
2. 审核 AI 生成的中英文翻译、别名、实体词和相关词
3. 配置排除词、地区、语言和来源范围
4. 每小时获取更新后的热点事件卡片
5. 查看事件热度、增速、摘要、来源和原始证据
6. 标记相关或不相关，改进后续评测与词项建议

## 功能需求

### 监控任务

每个监控任务包含：

- 名称和核心意图
- 一个或多个核心词
- 用户确认的翻译、别名、实体词和相关词
- 排除词
- 中英文语言范围
- 地区和数据来源范围
- 相关性阈值
- 每小时执行周期
- 启用或暂停状态

AI 扩展词未经用户确认不能扩大正式采集范围。低置信度扩展词只能作为建议展示。

### 热点事件卡片

事件卡片必须包含：

- 中英文事件标题
- 事件摘要
- 相关性分数和命中原因
- 当前热度、热度增速和趋势状态
- 首次发现、最近活跃和峰值时间
- 新闻、社交和视频来源分布
- 代表内容和全部有效证据链接
- 来源可信度和数据更新时间
- 面向创作者的关注理由

原始内容是事件卡片的证据，不是首页核心结果对象。

### 跨语言事件

系统必须识别中文和英文内容是否描述同一现实事件。事件聚合需要综合实体、动作、时间和多语言语义，不能只比较标题相似度。

同一事件的独立报道必须保留。转载、镜像和轻微改写内容可以折叠，但不能重复贡献热度。

### 热度与排序

不同来源先在各自平台内归一化，再计算事件热度。用户最终排序同时考虑：

- 监控相关性
- 当前热度
- 热度增速
- 来源可信度

全网很热但与监控意图无关的事件不能进入结果。

### 可追溯 AI

AI 可以执行：

- 双语扩词和实体辅助
- 事件标题与摘要生成
- 面向创作者的关注理由
- 复杂聚类边界的低频辅助判断

AI 输出必须引用有效的原始内容。AI 服务失败时，采集、匹配、聚类、热度和原始证据查询仍需工作。

## 数据来源边界

MVP 只使用：

- 官方 API
- RSS 或 Atom
- 获得授权的新闻 Feed

禁止绕过登录、验证码、反爬或平台访问限制。每个来源必须记录授权方式、商业使用限制、保留期限和删除要求。

## 容量与时效

MVP 按以下规模设计：

- 用户不超过 100 个
- 监控任务不超过 1,000 个
- 每日原始内容不超过 100,000 条
- 从内容可被 Connector 读取到事件卡片完成更新，P95 不超过 60 分钟

该规模不需要微服务、Kafka、Elasticsearch 或独立向量数据库。

## 非目标

MVP 不包含：

- Web 或 Miniapp 页面实现
- 秒级或分钟级实时流处理
- 无限制全网爬取
- 自动生成并发布完整文章或视频
- 对事件真实性作最终裁决
- 在线训练推荐模型
- 企业级多租户和计费系统
- 用户自定义评分公式或工作流

## 关键场景

```gherkin
Scenario: 创作者监控全球 AI Agent 热点
  Given 用户创建“AI Agent”监控任务
  And 用户确认中英文扩展词和排除词
  When 系统完成一次小时级采集与分析
  Then 用户看到按相关性和热度排序的事件卡片
  And 每张卡片包含摘要、趋势和原始来源
  And 中英文报道同一事件时只生成一个事件
```

```gherkin
Scenario: 单一来源不可用
  Given 一个社交平台返回限流或服务错误
  When 小时级流水线继续执行
  Then 其他来源仍完成采集和事件更新
  And 失败来源记录可重试状态
```

## 验收标准

- 人工标注集上热点结果 `Precision@20 >= 80%`
- 跨语言事件聚类 `F1 >= 75%`
- 95% 的可采集内容在 60 分钟内形成或更新事件卡片
- 重跑同一采集窗口不产生重复内容和重复事件证据
- 单一来源失败不阻塞其他来源
- 所有 AI 摘要能够追溯到有效内容
- 来源删除内容后不再参与展示、热度和摘要
- 所有 JSON API 使用统一 Result 响应

## 风险与边界

- 平台 API 配额可能限制覆盖范围，查询计划必须支持跨用户复用
- 跨语言聚类质量依赖标注集和阈值校准，模型升级必须回归验证
- 不同平台热度指标含义不同，禁止直接比较原始点赞或播放量
- 数据保留和删除规则以来源协议为准，业务需求不能覆盖合规要求
