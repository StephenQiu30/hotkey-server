# 011 知乎授权与采集

## 功能目标

支持用户连接自己的知乎授权能力，在授权范围内采集与主题相关的问题、回答、文章、专栏和公开互动指标。

## 用户故事

1. 作为研究者，我可以监控知乎上的深度讨论和问答趋势。
2. 作为用户，我可以基于关键词发现相关问题、回答和文章。
3. 作为用户，我希望知乎长文本能进入 AI 摘要和事件时间线。

## 范围

1. 用户授权连接、刷新和撤权。
2. 问题、回答、文章、专栏内容采集。
3. 标题、正文、作者、发布时间、链接、互动指标和问题上下文标准化。
4. 长文本截断、内容删除、权限限制和频控状态。
5. simulator fixture 和测试账号 smoke。

## 非范围

1. 自动回答、评论或私信。
2. 访问未授权私密内容。
3. 绕过反爬或风控限制。

## 状态模型

采集状态包含：`answer_available`、`article_available`、`content_truncated`、`permission_limited`、`item_deleted`。

## 契约影响

normalized content 必须支持长文本摘要输入边界和 question context 字段。平台能力必须返回长文本支持和截断策略。

## SDD 要求

1. 先定义问题、回答和文章的字段映射。
2. 先定义长文本截断和摘要输入策略。
3. 先定义删除、权限限制和频控的失败分类。

## TDD 要求

1. 先写回答、文章和问题 fixture 标准化测试。
2. 先写长文本截断不破坏摘要输入的测试。
3. 先写删除内容和权限限制测试。
4. 先写知乎 URL 与其他平台引用去重测试。

## E2E 验收门禁

1. simulator 返回回答、文章、删除项和长文本。
2. 长文本内容能进入 AI 摘要链路并生成事件关键信号。
3. 频控或权限限制在 Web 和日报中可见。

## 数据与权限

1. 凭据加密存储。
2. 长文本原文按 30 天留存。
3. 删除或不可访问内容不再进入 Obsidian 新笔记。
