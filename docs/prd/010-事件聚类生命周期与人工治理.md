# 事件聚类、生命周期与人工治理

## 目标

把相关 Content 聚合为唯一全局 Event，支持跨来源、跨语言、稳定键、生命周期、合并拆分和人工修正。

## 范围

- 实现 event 模块中的 Event、EventContent、MonitorEvent 和聚类决策。
- 实现时间、实体、标题、向量和来源特征的有限候选召回。
- 生成稳定 event_key、非唯一 event_fingerprint 和 fingerprint_version。
- 实现 detected、active、cooling、closed、archived、rejected、merged 等生命周期及迟到内容处理。
- 实现事务化合并、拆分、成员增删、证据删除和人工锁。
- 提供 Event 列表、详情、证据、合并、拆分和锁定 API。

## 非范围

- 不计算最终热度趋势。
- 不让 LLM 单独决定事件事实。
- 不物理删除合并历史、人工决策或聚类运行证据。

## 功能要求

1. 同一输入、特征和 clustering_version 重跑不创建重复 Event 或成员。
2. 确定性阈值处理高置信接受/拒绝，灰区才允许模型辅助。
3. event_key 在标题、摘要和成员变化时保持稳定。
4. 合并只在本 PRD 拥有的 `event_contents`、`monitor_events` 和 `merged_into_id` 上执行；实体、主张、Topic、Report 等关系通过 append-only `downstream_reconcile_required` 审计事件交给后续任务消费，并保留 merged_into_id。
5. 拆分创建新 Event、迁移选定成员并触发两侧重算。
6. 人工锁定字段和成员优先于后续算法任务。
7. 删除或过期证据后重新评估成员、状态和代表内容。
8. 候选召回必须由实体/动作、时间/地区、向量和事实指纹通道取并集，单条 Content 最多精排 50 个 Event；不得执行 Content×Event 全量比较。
9. `event_key` 由 IDGenerator 创建后不可变，`event_fingerprint` 只用于候选召回；同一 `content_id + clustering_version + feature_input_hash` 重跑只能产生一个成员决策。
10. 评分固定为 entity/action 0.30、semantic 0.30、temporal 0.20、location 0.10、source/context 0.10；硬实体、动作或时间冲突一票否决，LLM 只能处理 65–79.99 的灰区。
11. 合并、拆分、成员迁移、Monitor 关系和审计必须在一个 PostgreSQL 事务内完成，并按稳定 Event 键顺序锁行；失败不得留下半迁移关系。
12. `manual_locked` 的事件字段和成员关系优先于自动重算；证据删除、撤回和迟到内容必须产生可审计的重算或人工复核结果。
13. 所有 HTTP 接口使用统一 `Result`、numeric code、角色授权和 optimistic version；响应不得包含 Prompt、完整正文、Provider 原文、密钥或内部 SQL。查询端点为 `GET /api/v1/events`、`GET /api/v1/events/:id`、`GET /api/v1/events/:id/contents`；成员锁由 editor 通过 `POST /api/v1/events/:id/contents/:content_id/lock` 执行，body 为 `expected_version`、`locked`、`reason`；生命周期、合并、拆分由 admin 执行。merge 必须提交源版本与最终 canonical target 的版本：若提交的 target 已 merged，服务先解析 canonical root，再以 `target_expected_version` 比较 root；任一版本陈旧均返回 409。split 必须提交源事件版本和逐成员版本，每个写请求必须携带适用的 `expected_version`。

## 交付物

- Event 领域模型、聚类服务、Repository、事务用例和 API。
- 聚类特征、决策记录、版本配置与人工治理审计。
- Schema、记录模型、OpenAPI 和评测 Fixture。
- 合并、拆分、并发、重跑、删除证据和故障注入测试。

## 验收标准

- 固定跨语言聚类评测集 F1 不低于 75%，且每条输入候选数不超过 50。
- 无 LLM 时仍可创建、更新和关闭 Event。
- 合并/拆分任一事务失败不留下半迁移关系。
- 人工锁在后续聚类和重算后保持有效。
- 每个成员决策可追溯特征、算法版本、证据和决策者。
- 相同输入重跑不创建重复 Event 或成员，删除最后有效证据不会物理删除审计历史。
- 非管理员、过期版本、非法状态迁移和越权事件 ID 均被拒绝，并有稳定 HTTP/业务码证据。

## 完成定义

Event 成为全局唯一热点事实；后续热度、摘要、知识和报告均只引用 Event。

## 明确非范围

本 PRD 不实现热度趋势、实体/Claim、AI 摘要、River/cron 编排、知识库投影、报告投递或新增 AI `task_type`，也不直接迁移这些模块拥有的关系。这些能力只能由后续 PRD 消费 `downstream_reconcile_required` 审计事件及稳定 Event 键。
