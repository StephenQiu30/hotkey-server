# 007 YouTube 授权与采集

## 功能目标

支持用户授权 YouTube，并采集与主题相关的视频元数据、标题、简介、字幕/转写可用文本、频道信息、发布时间和互动指标。

## 用户故事

1. 作为用户，我可以监控某个关键词在 YouTube 上的新视频。
2. 作为用户，我可以选择频道或关键词作为采集范围。
3. 作为用户，我希望视频内容能被摘要为事件信号，而不是只看到链接。

## 范围

1. YouTube 授权连接、刷新和撤权。
2. 关键词、频道和视频搜索采集。
3. 视频标题、简介、字幕文本、频道、发布时间、互动指标标准化。
4. 字幕不可用、视频下架、地区限制和限流状态。
5. simulator fixture 和 live smoke。

## 非范围

1. 下载完整视频。
2. 绕过版权、地区限制或付费限制。
3. 自动评论或发布视频。

## 状态模型

视频文本状态：`transcript_available`、`transcript_missing`、`transcript_restricted`、`video_unavailable`。

## 契约影响

平台能力必须返回是否支持字幕文本、频道采集、互动指标和受限视频原因。normalized content 需要支持媒体内容类型。

## SDD 要求

1. 先定义视频内容映射和字幕文本长度上限。
2. 先定义字幕缺失时摘要输入策略。
3. 先定义地区限制、下架和限流失败分类。

## TDD 要求

1. 先写视频元数据标准化测试。
2. 先写字幕缺失降级测试。
3. 先写下架视频不进入事件聚类的测试。
4. 先写频道重复视频幂等测试。

## E2E 验收门禁

1. simulator 返回有字幕、无字幕、下架和限流视频。
2. 有字幕视频进入 AI 摘要链路。
3. 无字幕视频仍可基于标题简介生成弱信号并标注可信度较低。
4. 撤权后频道采集停止。

## 数据与权限

1. 不保存视频二进制。
2. 字幕文本按原文留存策略清理。
3. 版权或地区受限内容不做绕过采集。
