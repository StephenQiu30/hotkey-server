# 014 新闻与 RSS 采集

## 功能目标

支持系统新闻源、用户自定义新闻 RSS 和公开网页文章采集，用于补充社媒之外的事实信息和可信来源。

## 用户故事

1. 作为用户，我可以选择系统新闻源或添加自己的新闻 RSS。
2. 作为用户，我希望新闻源能提供更高可信度的事件背景。
3. 作为用户，我希望同一新闻被多个网站转载时不要重复出现在日报。

## 范围

1. 系统新闻源管理能力的产品要求。
2. 用户新闻 RSS 添加、校验和删除。
3. 公开网页文章抽取、标准化和去重。
4. 转载、canonical URL、语言识别和发布时间解析。
5. feed/webpage simulator fixture。

## 非范围

1. 付费墙绕过。
2. 未授权全文转载。
3. 新闻编辑后台。

## 状态模型

新闻源状态：`active`、`degraded`、`blocked`、`parse_failed`、`disabled`。

文章抽取状态：`full_text`、`partial_text`、`metadata_only`、`paywalled`、`unavailable`。

## 契约影响

OpenAPI 必须支持系统源列表、用户源列表、源健康状态和最近采集状态。标准化内容需要 source credibility hint 字段。

## SDD 要求

1. 先定义系统新闻源和用户新闻源边界。
2. 先定义正文抽取、canonical URL 和转载去重规则。
3. 先定义付费墙和不可访问内容降级规则。

## TDD 要求

1. 先写 RSS 和网页抽取 fixture 测试。
2. 先写 canonical URL 去重测试。
3. 先写付费墙降级测试。
4. 先写新闻可信度 hint 进入评分输入的测试。

## E2E 验收门禁

1. 系统新闻源产生文章并进入热点聚类。
2. 用户添加新闻 RSS 后产生采集任务。
3. 同一 canonical URL 不重复进入日报。
4. 付费墙内容标记为 metadata only，不进入全文摘要。

## 数据与权限

1. 公开网页原文按 30 天留存。
2. 付费墙和不可访问内容不做绕过。
3. 用户源删除后停止调度。
