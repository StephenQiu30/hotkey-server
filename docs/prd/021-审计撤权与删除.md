# 021 审计撤权与删除

## 功能目标

为平台授权、Git 授权、主题删除、账户删除、数据清理和关键任务失败提供可追踪审计，并保证用户撤权和删除操作真实生效。

## 用户故事

1. 作为用户，我可以撤销某个平台授权，并确认采集停止。
2. 作为用户，我可以删除主题或账户，并查看清理进度。
3. 作为运维，我可以定位失败任务，但不能看到用户明文凭据。

## 范围

1. 审计事件模型。
2. 授权撤权流程。
3. 主题删除和账户删除流程。
4. 数据库、搜索索引、队列任务和 MinIO 对象清理。
5. 清理失败重试和用户可见状态。

## 非范围

1. 企业合规报表。
2. 管理员导出用户完整行为日志。
3. 手工跳过删除流程。

## 状态模型

审计事件类型：`auth_connected`、`auth_refreshed`、`auth_revoked`、`topic_deleted`、`account_delete_requested`、`object_deleted`、`cleanup_failed`、`mail_sent`、`git_synced`。

清理状态：`queued`、`running`、`blocked_by_retry`、`partially_failed`、`completed`。

## 契约影响

OpenAPI 必须提供撤权接口、删除账户接口、主题删除接口、清理状态查询和用户可见审计摘要。内部任务必须能按用户和资源维度取消或阻止继续执行。

## SDD 要求

1. 先定义审计事件 schema 和敏感字段禁止清单。
2. 先定义撤权后的任务取消和凭据销毁顺序。
3. 先定义删除账户的数据清理拓扑。
4. 先定义部分失败后的重试、告警和用户提示。

## TDD 要求

1. 先写撤权后任务不可访问 token 测试。
2. 先写删除主题后日报和采集排除该主题测试。
3. 先写删除账户后数据库、索引、队列和 MinIO 清理测试。
4. 先写审计日志不包含敏感字段测试。
5. 先写部分失败可重试测试。

## E2E 验收门禁

1. 用户连接平台、创建主题并产生内容。
2. 用户撤权，后续采集停止且 Web 显示 revoked。
3. 用户删除主题，事件和日报候选不再包含该主题。
4. 用户删除账户，清理状态最终 completed。
5. 审计摘要能说明发生了什么，但不泄漏 token。

## 数据与权限

1. 审计保留必要行为和资源 ID，不保存明文凭据。
2. 清理任务必须幂等。
3. 删除请求不能被普通采集任务重新创建用户数据。
