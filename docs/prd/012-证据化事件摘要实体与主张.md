# 证据化事件摘要、实体与主张

## 目标

在不把 LLM 当作事实源的前提下，为 Event 生成结构化双语摘要、时间线、实体、关系和可引用主张。

## 范围

- 在 AI 运行基础上实现 event_summary、entity_claim_extraction 和 cluster_review 灰区任务。
- 实现 entities、entity_aliases、event_entities、entity_relations、claims、claim_evidences。
- 所有摘要句、时间线项和主张绑定有效 Content 与 evidence_locator。
- 实现 JSON Schema 校验、一次修复、复用失效和人工锁边界。
- 提供 Event 智能结果查询、人工修正和重新生成 API。
- LLM 不可用时生成基于代表 Content 的简化摘要。

## 非范围

- 不自动把 AI 输出写入 Vault。
- 不让模型覆盖人工锁、删除状态或事件成员事实。
- 不把未引用结论写入正式摘要或报告候选。

## 功能要求

1. 每个结构化输出保存 task_type、Schema、Prompt、模型、输入和证据版本。
2. 引用必须指向本次输入中的 active Content 和可定位片段。
3. 证据删除、事件合并拆分、人工锁或配置切换使旧结果失效。
4. 相同 reuse_key 的成功运行不重复调用 Provider。
5. 实体规范化保留别名与来源；主张保存支持、反对或中立立场。
6. 人工修正创建新版本和审计记录，不篡改历史 ai_run。
7. 全部 Provider 失败时 Event、证据和热度 API 仍可用。

## 交付物

- 事件智能用例、实体/主张领域模型、Repository 和 API。
- 任务 JSON Schema、Prompt 版本与证据定位校验。
- Schema、记录模型、OpenAPI 和固定评测集。
- 引用、幻觉、失效、人工锁、降级和 Provider 故障测试。

## 验收标准

- 摘要、时间线、实体和主张的核心结论都能定位到有效 Content。
- 非法引用和无法定位片段不能进入正式结果。
- 删除证据后旧结果不再被复用。
- 无 LLM 时返回明确降级摘要，不阻断结构化热点查询。
- 固定评测集记录结构通过率、引用正确率和幻觉率基线。

## 完成定义

PRD-013 可以把摘要任务作为可降级 Job；PRD-014/015 只能消费已验证结果和证据。
