# 多语言相关性匹配与反馈

## 目标

以有限候选召回、混合评分和可解释结果，把 active Content 匹配到正确 Monitor，并在模型不可用时保持规则降级链路。

## 范围

- 实现词法、实体和向量候选召回，禁止 Content 与全部 Monitor 笛卡尔比较。
- 实现核心词、AI 扩展词、排除词、冲突实体、语言和用户偏好的混合评分。
- 保存 monitor_matches、评分分项、证据、原因码、模型与 scoring_version。
- 对灰区结果可调用 relevance_review，确定性高低阈值直接接受或拒绝。
- 实现基于固定最近 Content 样本的无副作用相关性预览，返回命中、拒绝、解释和评分版本。
- 实现用户接受、拒绝、误报和漏报反馈，进入离线评测与词项建议。
- 提供 Monitor 匹配查询与解释 API。

## 非范围

- 不做在线训练或自动修改正式规则。
- 不创建 Event、热度或报告。
- LLM 不得覆盖排除词、冲突实体或人工锁定规则。

## 功能要求

1. 用户核心词权重高于 AI 扩展词。
2. 排除词和冲突实体可否决高向量相似度误报。
3. 中英文内容可匹配同一 Monitor 意图。
4. 每个接受/拒绝决策保存分项、证据、版本和最终决策者。
5. Embedding/LLM 不可用时继续执行词法与实体链路。
6. 同一 content、monitor、version 和 input hash 重跑不产生重复匹配。
7. 反馈不直接改变在线模型，只进入固定评测集和待审核建议。
8. 相关性预览不写 monitor_matches、Event 或下游 Job。

## 交付物

- 相关性领域服务、候选索引、Repository 和应用用例。
- 匹配解释、反馈和评测数据 API。
- Schema、记录模型、OpenAPI、评分版本与阈值配置。
- 中英 Fixture、排除/冲突、降级、幂等和性能测试。

## 验收标准

- 固定标注集 Precision@20 不低于 80%。
- 每条匹配均可解释其词法、实体、向量和规则贡献。
- 相同输入重跑只保留一个有效匹配。
- 候选召回数量有上限，目标容量下不出现全量比较。
- 所有 AI 不可用时仍能产生规则匹配和拒绝结果。

## 完成定义

PRD-010 只消费已接受且证据有效的匹配，不重新实现 Monitor 相关性规则。
