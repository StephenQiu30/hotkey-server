# 多语言相关性匹配与反馈

## 目标

以有限候选召回、混合评分和可解释结果，把 active Content 匹配到正确 Monitor，并在模型不可用时保持规则降级链路。

## 范围

- 实现词法、实体和向量候选召回，禁止 Content 与全部 Monitor 笛卡尔比较。
- 实现核心词、AI 扩展词、排除词、冲突实体、语言和用户偏好的混合评分。
- 保存 monitor_matches、评分分项、证据、原因码、模型与 scoring_version。
- 扩展 intelligence 的受控 relevance_review 任务：只在灰区调用，确定性高低阈值和硬否决永远优先；无可用 profile、预算或 Provider 时保留规则 review 结果。
- 实现基于固定最近 Content 样本的无副作用相关性预览，返回命中、拒绝、解释和评分版本。
- 实现用户接受、拒绝、误报和漏报反馈，进入离线评测与词项建议。
- 提供 Monitor 匹配查询与解释 API。

## 非范围

- 不做在线训练或自动修改正式规则。
- 不创建 Event、热度或报告。
- LLM 不得覆盖排除词、冲突实体或人工锁定规则。

## 功能要求

1. 用户核心词权重高于 AI 扩展词。
2. 排除词和冲突实体可否决高向量相似度误报。
3. 中英文内容可匹配同一 Monitor 意图。
4. 每个接受/拒绝决策保存分项、证据、版本和最终决策者。
5. Embedding/LLM 不可用时继续执行词法与实体链路。
6. 同一 content、monitor、version 和 input hash 重跑不产生重复匹配。
7. 反馈不直接改变在线模型，只进入固定评测集和待审核建议。
8. 相关性预览不写 monitor_matches、Event 或下游 Job。
9. relevance_review 是本 PRD 拥有的唯一 AI 任务扩展：只接受 OpenAI profile、静态 v1 输入输出 Schema 和固定 instruction；输入只含受限 Content 摘要、Monitor 意图、分项分数和原因码，输出只含 accepted/rejected/review、0–100 分数和受限原因码。它不得收到密钥、对象键、完整正文、Provider 原文或自由 Prompt。
10. 高置信排除词、冲突实体和 manual_locked 必须在任何 AI 调用前终止为 rejected 或保留人工结果；AI 只能处理 60 到高阈值之间的灰区，不能覆盖硬否决，也不能直接修改 Monitor 规则。
11. 匹配事实以 monitor_config_version、content、input_hash、scoring_version 去重；同一精确输入重试返回同一记录，新 input_hash 创建可审计的新快照。每个快照分别保存向量 embedding profile/version/model 出处和可空 relevance_review ai_run 出处，二者不得混用。
12. 反馈是每个用户对一个 Monitor+Content 的可更新事实，类型仅 relevant、irrelevant、false_positive、false_negative；所有创建和更新只使用 feedback 自身 version 乐观锁。它进入离线 P@20/误命中评测和待审核建议，不自动改写 Monitor draft、已发布规则、匹配决策、模型或其他用户偏好。
13. rule_score 始终保存五分项产生的确定性评分。灰区 AI 成功时，llm_score 与 final_score 必须都等于经 Schema 校验的 AI 分数，AI decision 必须与该分数的高/低阈值一致；否则结果为安全 rule review。列表和下游只按 final_score、decision、id 排序，解释保留确定性 rule_score 和 review_ai_run。

## 交付物

- 相关性领域服务、候选索引、Repository、应用用例和受控 relevance_review 适配。
- 匹配解释、无副作用预览、反馈、评测和待审核建议 API。
- 完整 Schema、记录模型、OpenAPI、评分/模型版本、升级回退手册与固定标注集。
- 中英 Fixture、排除/冲突、降级、幂等和性能测试。

## 验收标准

- 固定标注集 Precision@20 不低于 80%，明确排除词误命中率低于 2%。
- 每条匹配均可解释其词法、实体、向量和规则贡献。
- 相同 config/content/input/scoring 重跑只保留一个匹配快照；不同输入保留有版本的新快照。
- source、词法、向量候选分别最多 8、12、12 个，去重后最多 20 个；查询计划与计数测试证明不出现 Content×Monitor 比较。
- 所有 AI 不可用时仍能产生规则 accepted/rejected/review 结果；AI 只在允许的灰区调用并安全降级。
- 浏览、预览、反馈、评测和建议接口均使用统一 Result、角色授权、版本冲突与 OpenAPI；解释不返回 Prompt、完整正文、对象键、Provider 原文、凭据或内部 SQL。
- 升级必须把旧 monitor_matches 标记为 legacy-v1 事实而不伪造当前输入，回退使用 PLAN-009 专用受控手册和可丢弃 PostgreSQL fixture 演练。

## 完成定义

PRD-010 只消费已接受且证据有效的匹配，不重新实现 Monitor 相关性规则。PLAN-009 完成后不新增其他 AI task_type、事件、热度、训练、自动规则修改或后台编排。
