# 002 账号体系与用户授权托管

## 功能目标

为个人创作者和研究者提供云端 SaaS 账号体系，并安全托管用户主动授权的平台账号、Git 凭据和邮件订阅配置。该能力是所有采集、Obsidian 同步和日报发送的前置条件。

## 用户故事

1. 作为用户，我可以用邮箱注册和登录 HotKey。
2. 作为用户，我可以连接、查看、刷新、撤销第三方平台授权。
3. 作为用户，我可以知道每个授权连接是否有效、失效、受限或需要重新授权。
4. 作为用户，我可以删除账户并确认相关 token、任务、缓存和对象被清理。

## 范围

1. 邮箱注册、登录、登出、会话续期和密码重置。
2. 第三方平台授权连接的创建、刷新、状态展示、撤权和审计。
3. Git provider 凭据托管。
4. token、refresh token、cookie 或其他敏感凭据的加密存储。
5. 用户级权限边界和数据隔离。

## 非范围

1. 团队、多成员和组织权限。
2. 企业 SSO。
3. 代替平台官方授权规则绕过访问限制。
4. 管理员手动查看用户明文 token。

## 状态模型

用户状态：`pending_email_verification`、`active`、`locked`、`deleting`、`deleted`。

授权连接状态：`draft`、`connected`、`refreshing`、`limited`、`expired`、`revoked`、`failed`。

删除任务状态：`queued`、`running`、`partially_failed`、`completed`。

## 契约影响

Server OpenAPI 必须提供：

1. 账号注册、登录、登出、当前用户、密码重置接口。
2. 授权连接列表、详情、创建、回调、刷新、撤权接口。
3. 授权连接健康状态和最近失败原因字段。
4. 用户删除请求、删除状态查询接口。

Web 和小程序不得手写这些 API 类型。

## E2E 验收门禁

1. 用户完成邮箱注册登录。
2. 用户连接一个测试平台授权并看到 `connected`。
3. 测试平台返回 token 失效后，Web 显示 `expired` 和重新授权入口。
4. 用户撤权后，该平台采集任务停止并显示 `revoked`。
5. 用户删除账户后，相关授权、任务、缓存对象和索引全部进入清理流程。

## 数据与权限

1. 敏感凭据只能加密存储，不允许日志输出。
2. 每个用户只能访问自己的授权连接。
3. 审计记录保留必要事件，不保存明文凭据。
4. 删除账户必须触发异步清理并提供状态查询。
