# 013 微信公众号与 RSS 采集

## 功能目标

支持用户通过公众号公开链接、用户订阅源或 RSS 源采集文章内容，并纳入统一热点事件处理链路。

## 用户故事

1. 作为用户，我可以添加公众号文章链接或 RSS 源。
2. 作为用户，我可以把公众号/RSS 加入某个主题。
3. 作为用户，我希望长文章被摘要为事件笔记，而不是只保存链接。

## 范围

1. RSS 源添加、校验、更新和删除。
2. 公众号公开文章链接或可用订阅源采集。
3. 标题、正文、作者、发布时间、来源链接和封面引用标准化。
4. 源不可访问、解析失败、文章删除和重复文章处理。
5. feed simulator fixture。

## 非范围

1. 采集未公开公众号后台数据。
2. 绕过微信访问限制。
3. 公众号自动群发或管理。

## 状态模型

源状态：`active`、`invalid_url`、`fetch_failed`、`parse_failed`、`stale`、`disabled`。

文章状态：`new`、`duplicate`、`content_extracted`、`content_partial`、`unavailable`。

## 契约影响

OpenAPI 必须支持用户 RSS 源管理、源健康状态、最近抓取结果和解析失败原因。normalized content 必须支持 article 类型。

## SDD 要求

1. 先定义 RSS 源 schema 和抓取频率上限。
2. 先定义文章正文抽取和失败分类。
3. 先定义公众号公开链接的合规边界和降级策略。

## TDD 要求

1. 先写 RSS 解析和 invalid feed 测试。
2. 先写文章正文抽取失败降级测试。
3. 先写重复 GUID、重复 URL 和标题近似重复测试。
4. 先写源删除后不再调度测试。

## E2E 验收门禁

1. 用户添加测试 RSS 源并看到健康状态。
2. 新文章进入标准化、相似度和事件聚类链路。
3. RSS 源失效时 Web 显示失败原因，日报标记源失败。
4. 删除 RSS 源后后续不再采集。

## 数据与权限

1. 用户自定义 RSS 源只属于当前用户。
2. 原文快照写入 MinIO 并按生命周期清理。
3. 解析失败只保存必要错误和源健康信息。
