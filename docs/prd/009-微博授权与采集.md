# 009 微博授权与采集

## 功能目标

支持用户连接自己的微博授权能力，在授权范围内采集与主题相关的微博内容、作者、发布时间、链接和公开互动指标。

## 用户故事

1. 作为用户，我可以把微博加入国内热点主题监控。
2. 作为用户，我可以看到微博连接是否失效、受限或需要重新授权。
3. 作为用户，我希望微博热点能与新闻、知乎、B站等中文内容聚合。

## 范围

1. 用户授权连接、刷新和撤权。
2. 关键词相关内容采集。
3. 微博正文、作者、发布时间、链接、转评赞指标和媒体引用标准化。
4. 登录失效、频控、内容删除和受限可见状态。
5. simulator fixture 和测试账号 smoke。

## 非范围

1. 自动发微博、评论或私信。
2. 采集用户未授权的私密内容。
3. 绕过平台风控。

## 状态模型

采集状态包含：`login_required`、`visibility_limited`、`item_deleted`、`frequency_limited`。

## 契约影响

平台能力需要暴露中文内容、互动指标、媒体引用、授权失效原因和用户可操作的重新授权入口。

## SDD 要求

1. 实现前核验微博当前授权和合规接入边界。
2. 先定义微博字段映射、中文分词输入和 URL 规范化规则。
3. 先定义频控后的退避策略和用户提示。

## TDD 要求

1. 先写微博 fixture 标准化测试。
2. 先写中文关键词和排除词命中测试。
3. 先写频控失败分类测试。
4. 先写微博删除内容不进入日报测试。

## E2E 验收门禁

1. simulator 完成授权、采集、标准化、聚类和日报链路。
2. 测试账号 smoke 验证最小授权和采集路径。
3. 频控时 Web 显示微博受限，不影响其他平台。

## 数据与权限

1. 授权凭据加密存储。
2. 原文和媒体引用进入 MinIO 生命周期清理。
3. 用户撤权后微博采集任务停止。
