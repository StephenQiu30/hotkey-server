# 身份、认证、会话与权限

## 目标

交付可供全部业务 API 复用的用户、密码认证、可撤销会话和 admin/editor/viewer 授权能力。

## 范围

- 实现 identity 模块四层结构。
- 实现 users、user_preferences、auth_sessions 的记录模型、Repository 和应用用例。
- 支持注册、登录、刷新、退出、当前用户查询、密码修改和用户停用。
- 使用 bcrypt、JWT v5 和 PostgreSQL 会话事实；Redis 只能作为可选缓存或限流。
- 建立角色与资源级授权中间件，审计敏感操作。
- 发布 /api/v1/auth 和安全的 /api/v1/users 契约。

## 非范围

- 不实现第三方 OAuth、企业 SSO、多租户或前端页面。
- 不开放任意用户表通用 CRUD。
- 不把 JWT 作为不可撤销的唯一会话事实。

## 功能要求

1. 邮箱规范化后唯一，密码只保存 bcrypt 哈希。
2. 访问令牌短期有效，刷新必须校验活动会话、用户状态和令牌轮换。
3. 退出、密码修改和用户停用可以撤销相关会话。
4. admin 管理用户；editor 可维护业务内容；viewer 只读允许资源。
5. 认证与授权失败使用统一 Result 和稳定数字业务码。
6. 登录、刷新、退出和高风险管理操作写入审计日志。
7. 并发刷新只允许一个有效结果，旧刷新凭据不可重复使用。

## 交付物

- identity 领域、应用、PostgreSQL 适配器和 HTTP 路由。
- Schema、记录模型、OpenAPI 和错误码更新。
- 密码、JWT、会话轮换、撤销、RBAC 和并发测试。
- 可用于其他模块的认证主体与授权接口。

## 验收标准

- 注册、登录、刷新、退出、停用完整集成链路通过。
- 无效密码、过期令牌、撤销会话、角色不足返回正确 HTTP 和业务码。
- 业务 JSON 不包含 request_id、密码哈希、会话密钥或内部错误。
- PostgreSQL/Redis 组合和无 Redis 两种模式均不改变业务正确性。
- 所有 identity 表具备 Repository 契约和并发更新测试。

## 完成定义

PRD-005 及之后的公共业务 API 必须使用本任务提供的主体和授权能力。
