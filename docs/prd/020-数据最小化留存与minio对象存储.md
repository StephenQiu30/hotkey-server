# 020 数据最小化留存与 MinIO 对象存储

## 功能目标

落实最小化留存策略，并使用 MinIO 存储原始内容快照、采集附件引用、导出产物、邮件归档副本和可追踪处理对象。

## 用户故事

1. 作为用户，我知道 HotKey 不长期保存不必要原文。
2. 作为用户，我可以删除账户并清理相关对象。
3. 作为运维，我可以通过生命周期策略自动清理过期对象。

## 范围

1. MinIO bucket 和对象分类要求。
2. 原文默认 30 天留存。
3. 长期保留事件摘要、评分、同步记录和必要审计。
4. 对象元数据、过期时间和删除任务。
5. 对象访问权限和审计。

## 非范围

1. 无限期保存全部原始内容。
2. 把 MinIO 当作用户可浏览网盘。
3. 存储未经授权的视频、图片二进制。

## 状态模型

对象状态：`active`、`scheduled_for_expiry`、`expired`、`delete_pending`、`deleted`、`delete_failed`。

留存策略：`raw_30_days`、`short_cache_7_days`、`derived_long_term`、`audit_required`。

## 契约影响

内部对象引用必须包含 owner、content type、source platform、retention policy、expires at、checksum 和 delete status。OpenAPI 删除状态可返回对象清理进度摘要。

## SDD 要求

1. 先定义 bucket、prefix、元数据和生命周期策略。
2. 先定义哪些对象能长期保存，哪些必须过期。
3. 先定义删除账户、删除主题、撤权后的对象清理规则。
4. 先定义对象访问鉴权和短期签名 URL 策略。

## TDD 要求

1. 先写对象元数据和留存策略测试。
2. 先写过期对象清理任务测试。
3. 先写用户删除触发对象删除测试。
4. 先写跨用户不可访问对象测试。
5. 先写 MinIO 不可用时任务可重试测试。

## E2E 验收门禁

1. 采集原始快照写入测试 MinIO。
2. 对象元数据包含正确用户、平台、过期时间。
3. 生命周期或清理任务删除过期对象。
4. 用户删除账户后对象清理状态可查询。
5. MinIO 暂时不可用时系统显示可恢复失败，不丢失任务。

## 数据与权限

1. 默认原文留存 30 天。
2. token 不允许写入 MinIO。
3. 跨用户对象访问必须拒绝。
4. 删除失败必须可观测并可重试。
