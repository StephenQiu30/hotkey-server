# 019 邮件日报周报

## 功能目标

按用户配置的时区、发送时间和主题范围发送固定模板日报和周报，帮助用户定期查看热点事件、Obsidian 同步结果和失败提醒。

## 用户故事

1. 作为用户，我可以开启或关闭日报、周报。
2. 作为用户，我可以配置发送时间、时区和包含主题。
3. 作为用户，我可以在邮件中看到热点列表、摘要、评分、来源和 Obsidian commit 链接。
4. 作为用户，我希望采集或同步失败时邮件明确提示。

## 范围

1. 日报/周报订阅配置。
2. 固定邮件模板。
3. 事件选择、排序和去重。
4. 邮件发送队列、重试和幂等。
5. 邮件发送状态、失败原因和审计。

## 非范围

1. 用户自定义邮件模板。
2. 即时热点告警。
3. 短信、飞书、企业微信等通知渠道。

## 状态模型

订阅状态：`enabled`、`disabled`、`invalid_email`、`paused_due_to_bounce`。

发送任务状态：`scheduled`、`rendering`、`sending`、`sent`、`retrying`、`failed`、`skipped_no_content`。

## 契约影响

OpenAPI 必须提供订阅配置、测试发送、发送历史、失败原因和最近邮件预览。事件接口需要支持日报候选查询。

## SDD 要求

1. 先定义邮件模板字段。
2. 先定义日报和周报事件选择规则。
3. 先定义时区、幂等、重试和退订/暂停规则。
4. 先定义发送失败对 Web 状态和下一次调度的影响。

## TDD 要求

1. 先写时区调度测试。
2. 先写邮件模板渲染快照测试。
3. 先写无内容跳过测试。
4. 先写发送失败重试和幂等测试。
5. 先写 Obsidian commit 链接进入邮件测试。

## E2E 验收门禁

1. 用户配置日报时间并触发测试发送。
2. 邮件包含事件摘要、评分、来源和 commit 链接。
3. 同一日报不会重复发送。
4. 邮件 provider 失败后进入 retrying，并在 Web 显示。
5. 用户关闭订阅后不再发送。

## 数据与权限

1. 邮件地址属于用户敏感数据。
2. 邮件归档副本如写入 MinIO，必须按留存策略清理。
3. 邮件不得包含平台 token 或内部错误堆栈。
