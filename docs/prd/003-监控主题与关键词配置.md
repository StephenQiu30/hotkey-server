# 003 监控主题与关键词配置

## 功能目标

让用户定义要监控的热点主题，包括关键词、排除词、语言、平台范围、相似度阈值、采集频率、日报范围和 Obsidian 输出目录。

## 用户故事

1. 作为用户，我可以创建一个主题，例如“AI Agent 商业化”。
2. 作为用户，我可以配置关键词、同义词、排除词和语言范围。
3. 作为用户，我可以选择要监听的平台和账号连接。
4. 作为用户，我可以暂停、恢复、复制和删除主题。

## 范围

1. 主题 CRUD。
2. 关键词、排除词、语义描述、语言和地区配置。
3. 平台选择和授权连接绑定。
4. 相似度阈值、热度阈值和采集频率配置。
5. 主题级日报/周报订阅开关。

## 非范围

1. 团队共享主题。
2. 复杂布尔查询语言。
3. 用户自定义算法权重界面。
4. 自动生成创作稿件。

## 状态模型

主题状态：`draft`、`active`、`paused`、`invalid`、`deleting`、`deleted`。

配置校验状态：`valid`、`missing_keyword`、`missing_source`、`invalid_threshold`、`auth_required`。

## 契约影响

OpenAPI 必须提供主题列表、详情、创建、更新、暂停、恢复、删除和配置校验接口。主题详情必须返回关联平台、授权连接状态、最近采集时间、最近事件时间和配置错误。

## TDD 要求

1. 先写主题创建和配置校验测试。
2. 先写暂停主题后不再调度采集的测试。
3. 先写删除主题后任务、缓存和事件引用处理测试。
4. 先写关键词、排除词和相似度阈值边界测试。

## E2E 验收门禁

1. 用户在 Web 创建主题并绑定至少一个平台。
2. 主题配置错误时不能启动采集，Web 显示具体原因。
3. 主题激活后产生采集任务。
4. 用户暂停主题后采集任务停止。
5. 用户删除主题后，后续日报不再包含该主题。

## 数据与权限

1. 主题属于单个用户。
2. 主题删除必须遵循最小化留存策略。
3. 配置历史只保存必要审计信息，不保留敏感 token。
