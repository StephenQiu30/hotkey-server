# 监控主题、规则与来源配置

## 目标

交付 Monitor 的 P0 控制平面：团队可协作编辑草稿，管理员能原子发布、暂停、恢复和归档，后续采集只读取不可变的已发布配置。来源连接作为全局安全资源管理；预览只解释配置、规则、查询签名和估算，不执行外部请求或 Content 匹配。

## 范围

- 建立 `monitor`、`source`、`operations` 三个四层模块，并保留 identity 提供的认证主体和角色检查边界。
- 将 Monitor 逻辑身份与 `monitor_config_versions` 分离；规则和来源关联属于配置版本，已发布或 superseded 版本及其子记录不可变。
- 实现 `draft/active/paused/archived` 状态机、版本前置条件、首次从已发布版本创建草稿的并发规则、发布回滚和历史归属约束。
- 实现 `rss`（RSS/Atom Feed）和 `hacker_news` SourceConnection 的 P0 配置、凭据引用校验、生命周期、执行语义冻结、SSRF 输入校验与安全 DTO。
- 实现规则矩阵、AI 候选的管理员审批、canonical `config_hash`/`query_signature` 与不持久化的配置预览。
- 实现 Result/OpenAPI 覆盖的 `/api/v1/monitors` 与 `/api/v1/source-connections` 业务接口、固定 `30000–30004` 与 `40000–40003` 业务码，以及 transactions 内的 operations audit 写入。
- 同步完整 `db/schema.sql`、数据库记录模型、Schema/架构测试、OpenAPI 和长期 Acceptance-005 证据。
- 仅同步 `collection_run_targets` 的目标 Schema/记录归属约束，使 future immutable MonitorSource 可被安全引用；不在本任务创建、调度、执行或推进任何 collection run/checkpoint。

## 非范围

- 不实现 Connector、真实 Fetch/Validate/Health、SSRF 出网防护执行、River Job、Cron、collection run/checkpoint 的应用或 Repository 行为、Content 标准化、Content 匹配、Embedding、事件或报告。
- 不实现 Content 命中/拒绝/解释预览；该能力属于 PRD-009。
- 不开放任意表 CRUD、动态 Connector 插件、通用规则 DSL、通用 QueryPlan 或秘密存储后端。
- 不实现多租户、Monitor owner/可见性或成员表；Monitor 是团队共享资源。

## 功能需求

1. **逻辑 Monitor 与版本事实。** 创建 Monitor 时创建一个 draft config。active/paused Monitor 的编辑只能创建或替换 draft，不得改写已发布配置；发布原子地将 draft 标记 published、旧 published 标记 superseded、推进 pointers 和 aggregate version，并写入 audit。发布失败、任一子记录写失败或 audit 失败必须整体回滚。
2. **并发与状态。** 修改既有 Monitor/Source 的写命令使用 Design-014 指定的 `expected_monitor_version`、`expected_draft_version` 或 `expected_source_version`，不使用通用版本字段；create 命令不传期望版本。两个并发 publish、两个并发首次创建 draft 都只能有一个成功。状态只能按 `draft → active`、`active → paused`、`paused → active`、`draft/active/paused → archived`、`archived → paused` 转换；过期版本固定返回 `30001`/409。
3. **配置校验。** 实现时区、语言、地区、频率、阈值、保留期、规则数和来源数限制；对每个 rule type/operator 执行固定兼容矩阵、长度、RE2 编译与审批规则。至少一个 approved、enabled、非 AI 核心规则与一个可调度来源是发布前提。
4. **权限。** viewer 只读已发布 Monitor 与安全来源 DTO；editor 可创建/编辑任意未归档 Monitor 草稿并做预览；admin 才能发布、状态切换、AI 候选审批和管理来源连接。不得以 `created_by` 或 `updated_by` 推断资源所有权。
5. **SourceConnection。** P0 仅允许 RSS/Atom 与 Hacker News；auth 类型、`env:NAME` credential reference、endpoint、config 白名单和默认值必须按 Design-014 验证。任何 published config 引用后，`source_type`、endpoint 与 config 不可更新；disable/archive 不得破坏 active Monitor 的唯一可调度来源；resume 必须拒绝零可调度来源。
6. **签名与预览。** 将 approved/enabled 规则与有效来源配置 canonicalize 后计算稳定 `query_signature`；未批准 AI 规则不参与它。完整 `config_hash` 保留规则的安全状态。preview 不读取 Content、Connector、Redis 或 River，且不写业务、版本、审计、checkpoint、run 或队列表。
7. **历史与安全。** Schema 用外键、部分唯一索引和触发器保护 published/superseded 不可变性、Monitor/Config/Source 历史归属和来源执行语义。HTTP、audit、日志与错误响应不得泄露 credential reference；viewer/editor 不能读 endpoint/config，admin 只能读经白名单验证的非秘密 endpoint/config。
8. **下游稳定输入。** 本任务只保存已发布 config、`query_signature`、`published_at` 和 immutable monitor source 事实；PLAN-006 将另行实现 shared `collection_runs`、targets、Connector 和 checkpoint 运行行为，不得在本任务提前实现。

## 交付物

- 版本化 Monitor、Rule、SourceConnection 和 Operations Audit 的领域、应用、PostgreSQL 与 HTTP 实现。
- 完整 Schema、记录模型、错误码、OpenAPI、bootstrap 装配和架构门禁。
- 单元、真实 PostgreSQL 集成与 HTTP 契约测试，覆盖正常、失败、权限、并发、幂等、归档/恢复和安全边界。
- 预先建立的 Acceptance-005 骨架，并在实现完成后写入受控 RED、绿灯、环境、提交范围、独立复审和残余风险证据。

## 验收标准

| 领域 | 必须证明的结果 |
|---|---|
| Schema 与历史 | 空库和二次执行通过；版本 pointers、部分唯一约束、复合外键与 immutable trigger 拒绝非法写入；记录模型、Schema 和 Repository 一致。 |
| 草稿与发布 | create、已有草稿替换、首次草稿、publish、publish rollback 和 concurrent publish 符合 version 协议；已发布配置不可改变。 |
| 状态与来源 | 完整合法/非法转换、pause/archive 排除 active-only 查询、restore→paused、resume 的可调度来源检查、source disable/archive 竞态正确。 |
| 规则与签名 | 矩阵、上限、regex、AI 审批、canonical hash、query override、语言/地区交集和 Source defaults 都可复现；未批准 AI 词不进入 query signature。 |
| 预览 | 预览结果稳定、估算可计算，调用前后所有 Monitor/Source/audit/run/River 事实无变化，且 fake Connector 证明没有外部调用。 |
| 权限与 HTTP | 401/403/成功完整矩阵；Result、稳定错误码、cursor、OpenAPI 与 DTO 脱敏正确；endpoint/config 只可由 admin management DTO 读取。 |
| 审计与安全 | 所有写动作与 audit 在同一 PostgreSQL 事务；失败完整回滚；HTTP、audit、Zap 测试均不含 endpoint 中的敏感片段、config secret 或 credential reference。 |
| 回归 | 真实 PostgreSQL 下的定向集成、OpenAPI 生成/校验、架构校验和 `make ci` 通过，构建产物清理且工作树干净。 |

## 完成定义

完成后，PLAN-006 可以只通过 Monitor/Source 的只读应用端口读取 immutable published config、来源执行语义、`query_signature` 和 `published_at`，无需重新解释任何 draft 或 HTTP DTO；但该下游 Plan 仍须自行实现采集运行事实。
