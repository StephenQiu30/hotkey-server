# 监控主题、规则与来源配置

## 目标

让用户能够安全定义、预览、发布、暂停和归档一个版本化 Monitor，为采集与匹配提供稳定输入。

## 范围

- 实现 monitor 和 source 配置部分的四层结构。
- 支持 monitors、monitor_rules、monitor_sources、source_connections 的安全业务操作。
- 支持核心词、排除词、实体约束、语言、地区、来源、频率和阈值配置。
- 实现 draft、active、paused、archived 状态与 version 乐观锁发布。
- 支持来源凭据引用、健康状态和不泄密的配置查询。
- 提供基于固定历史 Content 的无副作用规则预览接口。

## 非范围

- 不执行真实来源采集。
- 不实现通用规则 DSL、动态 Connector 插件或任意表 CRUD。
- AI 扩展词只保存审批状态，本任务不负责模型生成。

## 功能要求

1. 只有 active Monitor 可以进入调度；paused 和 archived 不产生新采集。
2. 规则、来源和频率以同一 Monitor 版本事务化发布。
3. 只有 approved 的 AI 候选词可进入正式查询与匹配。
4. 预览返回命中、拒绝、评分解释和估算请求量，不写正式匹配、事件或 Job。
5. 正则和输入长度有明确上限，拒绝高风险表达式。
6. 角色权限区分查看、编辑、发布、暂停和归档。
7. SourceConnection 只暴露凭据引用和健康摘要，不返回密钥。

## 交付物

- Monitor/Source 配置领域模型、Repository、应用用例和 HTTP API。
- Schema、记录模型、OpenAPI、审计事件和错误码更新。
- 状态机、乐观锁、权限、预览无副作用和密钥脱敏测试。

## 验收标准

- 并发发布只有一个版本成功，冲突返回 409。
- paused Monitor 不再被查询为到期对象，历史 Event 保持可读。
- 预览执行前后正式业务表与 River 队列无变化。
- 非管理员无法修改全局来源凭据或可信度配置。
- 所有 Monitor 配置变更可通过审计日志追溯。
