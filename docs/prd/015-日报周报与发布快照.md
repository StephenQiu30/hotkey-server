# 日报、周报与发布快照

## 目标

按用户时区和订阅范围生成可追溯、发布后不漂移的日报与周报，并投影到 Vault。

## 范围

- 实现 report 模块、reports、report_items 和报告生成状态。
- 支持 daily、weekly，按 Monitor/Topic/全局范围和用户时区确定周期。
- 从冻结的 Event、热度、趋势、摘要和证据生成结构化报告条目。
- 实现 report_draft AI 任务与无 LLM 的结构化降级报告。
- 保存每个 report_item 的 Event 版本、分数版本、摘要版本和证据快照引用。
- 发布报告到固定 Vault 目录，并提供列表、详情、预览、生成和重新生成 API。
- 通过 River 实现 build_report 的唯一任务和失败恢复。

## 非范围

- 不发送邮件或提供订阅 Feed。
- 不在报告生成时重新计算热度或修改 Event。
- 不允许已发布报告随 Event 后续变化而静默更新。

## 功能要求

1. report_type + scope + period + version 构成稳定幂等键。
2. 周期边界按订阅时区计算，数据库统一保存 UTC。
3. 报告至少包含摘要、重点事件、上升最快、持续关注、争议与缺口、覆盖说明。
4. 核心结论必须引用有效 Content 或冻结证据快照。
5. 草稿可重新生成；发布后内容不可原地漂移，只能创建新版本。
6. LLM 失败时仍生成结构化事件列表和来源覆盖说明。
7. Vault 写入复用 PRD-014 的原子写入与冲突规则。

## 交付物

- Report 领域、选择器、构建、发布和查询 API。
- 报告模板、冻结快照、Schema、记录模型和 OpenAPI。
- build_report Job、指标和管理员重试入口。
- 时区、周期、幂等、快照、证据、降级和发布故障测试。

## 验收标准

- 同一范围和周期重跑不产生重复有效报告。
- 已发布报告在 Event 更新、合并或热度重算后内容保持不变。
- 所有核心结论可定位到冻结证据。
- 日报、周报路径与 Frontmatter 稳定。
- LLM 不可用时仍能发布可读的结构化报告。

## 完成定义

PRD-016 只投递已发布的不可变报告版本，不直接读取变化中的 Event。
