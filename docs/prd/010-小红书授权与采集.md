# 010 小红书授权与采集

## 功能目标

支持用户连接自己的小红书授权能力，在授权范围内采集与主题相关的公开笔记文本、作者、发布时间、链接、标签和互动指标。

## 用户故事

1. 作为创作者，我可以监控小红书上的消费、生活方式和行业趋势。
2. 作为用户，我可以看到小红书采集是否受限或需要重新授权。
3. 作为用户，我希望小红书笔记与微博、知乎、B站和新闻合并成事件。

## 范围

1. 用户授权连接、状态检测和撤权。
2. 关键词相关公开笔记采集。
3. 标题、正文、标签、作者、发布时间、链接、互动指标标准化。
4. 内容不可见、风控、频控和 schema 变化处理。
5. simulator fixture 和测试账号 smoke。

## 非范围

1. 自动发布笔记或评论。
2. 私信、收藏夹和非授权内容。
3. 绕过平台限制。

## 状态模型

采集状态包含：`auth_required`、`risk_limited`、`frequency_limited`、`note_unavailable`、`schema_changed`。

## 契约影响

平台能力需要明确小红书是否处于可采集、受限或仅手动导入状态。若外部限制导致不可稳定采集，用户界面必须展示受限原因。

## SDD 要求

1. 实现前核验小红书当前授权和合规接入边界。
2. 先定义笔记字段映射和媒体引用处理。
3. 先定义风控触发后的停止、退避和告警策略。

## TDD 要求

1. 先写笔记标准化测试。
2. 先写标签与关键词匹配测试。
3. 先写风控受限状态测试。
4. 先写 schema 变化导致 provider_schema 失败分类测试。

## E2E 验收门禁

1. simulator 覆盖正常笔记、不可见笔记、频控和 schema 变化。
2. 受限状态在 Web 可见，并在日报中标记该平台未完整采集。
3. 撤权后任务停止且审计记录完整。

## 数据与权限

1. 凭据加密存储。
2. 媒体不下载原图，只保存必要引用和文本快照，除非后续 design 明确授权。
3. 原文快照按 30 天清理。
