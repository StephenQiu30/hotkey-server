# 015 内容标准化去重与相似度筛选

## 功能目标

将多平台采集内容转换为统一内容模型，并根据用户主题进行关键词过滤、语义相似度筛选、重复内容合并和低质量内容剔除。

## 用户故事

1. 作为用户，我只想看到与主题高度相关的内容。
2. 作为用户，我不希望同一链接或同一帖子的重复采集污染日报。
3. 作为用户，我希望中文、英文和跨语言内容都能进入同一主题判断。

## 范围

1. 平台内容标准化。
2. 语言识别、正文清洗、URL 规范化。
3. 关键词和排除词过滤。
4. embedding 相似度筛选。
5. exact duplicate、canonical URL duplicate、near duplicate 合并。
6. 质量分和可摘要性标记。

## 非范围

1. 事件级聚类和热度评分。
2. 复杂人工标注平台。
3. 用户可编辑 embedding 模型参数。

## 状态模型

内容处理状态：`raw_received`、`normalized`、`filtered_out`、`similarity_matched`、`duplicate`、`ready_for_clustering`、`processing_failed`。

去重类型：`source_id`、`canonical_url`、`text_hash`、`near_duplicate`。

## 契约影响

内部内容契约必须输出相似度分数、过滤原因、去重主记录、语言、质量分、原始对象引用和可摘要性标记。OpenAPI 事件详情可暴露来源内容列表和过滤摘要。

## SDD 要求

1. 先定义 normalized content schema。
2. 先定义主题匹配输入和输出字段。
3. 先定义去重键、相似度阈值和过滤原因枚举。
4. 先定义 embedding provider 抽象和失败降级策略。

## TDD 要求

1. 先写多平台 fixture 标准化测试。
2. 先写关键词、排除词和语言过滤测试。
3. 先写 exact/canonical/near duplicate 测试。
4. 先写 embedding 失败后的可恢复状态测试。
5. 先写跨语言相似内容通过阈值测试。

## E2E 验收门禁

1. provider simulator 返回多平台重复内容，系统只保留一个主记录。
2. 与主题无关内容被过滤且记录过滤原因。
3. 相关内容进入事件聚类。
4. embedding provider 临时失败时任务可重试，Web 不显示空白结果。

## 数据与权限

1. embedding 输入不得包含用户 token。
2. 过滤掉的原文仍按最小化留存清理。
3. 用户删除主题后相关匹配索引清理。
