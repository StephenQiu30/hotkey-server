## 排障流程

403：

1. 确认请求带 `Authorization: Bearer <token>`。
2. 查询用户角色：`select id,email,role,is_active from users where id=<id>;`
3. 对照上表确认路由所需权限。
4. 需要临时恢复时执行：`update users set role='admin' where id=<id>;`

401/token 异常：

1. 检查错误内容：`Missing access token`、`Invalid token signature`、`Token expired`。
2. 确认 `JWT_SECRET_KEY` 未在部署中被意外改变。
3. 用户重新登录刷新 token。
4. 若用户被停用，执行：`update users set is_active=true where id=<id>;`
