## 预定验收环境

- 可丢弃 PostgreSQL：`HOTKEY_TEST_DSN='postgres:///hotkey_plan005_test?sslmode=disable'`；运行时验证会重建 public schema，禁止指向生产数据库。
- Go、PostgreSQL、pgvector/pg_trgm 和 Git commit SHA 在验收时如实记录；不使用真实来源凭据、真实 Connector、真实 RSS/HN 请求或真实秘密。
- PLAN-005 不依赖 Redis、River 或 Content 数据；若任何验收隐式连接外部网络，视为越界并失败。

## 受控 RED 证据模板

每项 RED 必须只在未提交临时补丁中发生、运行到预期失败、立即反向恢复，并在恢复后运行对应 GREEN、`git diff --check`、`git status --short`。验收完成时填写实际命令、失败信号和恢复后的提交 SHA。

| 验收项 | 临时破坏 | RED 命令与预期失败 | 恢复后的 GREEN |
|---|---|---|---|
| 已发布配置不可变 | 移除 published rule/source trigger 检查 | 目标 PostgreSQL 集成测试应允许非法更新而失败 | Trigger 与集成测试重新拒绝更新。 |
| 发布乐观锁 | 跳过 expected monitor/draft version 条件 | 并发 publish 测试出现两个成功 | 恰好一个成功，另一个 `30001`/409。 |
| 发布事务审计 | 让 AuditWriter 返回错误 | 发布错误后仍有 pointer/version/child 写入 | 全部业务与 audit 事实回滚。 |
| 预览无副作用 | 使 preview 调用 write/audit path | 表快照或 fake external call 发生变化 | 所有业务/audit/run/River 事实不变，call count 为零。 |
| 来源敏感信息 | 将 credential reference 放入 DTO/audit metadata | HTTP/审计脱敏测试不再拒绝字段 | viewer/editor/admin 输出重新满足字段白名单。 |
| 角色边界 | 取消 admin 角色中间件或 application check | viewer/editor 可执行发布或来源管理 | 403 和不变的数据库事实恢复。 |

## 预定 GREEN 与复现命令

实现完成后至少执行并记录摘要：

```bash
HOTKEY_TEST_DSN='postgres:///hotkey_plan005_test?sslmode=disable' make ci
HOTKEY_TEST_DSN='postgres:///hotkey_plan005_test?sslmode=disable' \
  go test -race ./internal/modules/monitor/... ./internal/modules/source/... ./internal/modules/operations/... -count=1
make clean
git diff --check
git status --short
```

此外必须记录以下定向真实 PostgreSQL 证据：

1. published config/source/rule trigger、Monitor/config/source 复合外键、partial unique index 与 Schema 二次执行。
2. 两个 concurrent publish 和两个首次 draft 创建各恰有一个成功；rollback 不留 config pointer、child 或 audit 残留。
3. 完整状态机、active-only published query、source disable/archive、resume 零来源拒绝与 archive/restore 行为。
4. Rule matrix、RE2、AI 审批、Source defaults、canonical `config_hash`/`query_signature`、query override 和 locale intersection。
5. preview 前后 Monitor/Source/audit/collection/River 表快照一致，并以 fake port 证明零外部访问。
6. viewer/editor/admin HTTP 矩阵、Result `data:null` error、业务码、cursor、OpenAPI 和 DTO/日志/audit 脱敏。
