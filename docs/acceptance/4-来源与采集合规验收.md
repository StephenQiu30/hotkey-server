# 4-来源与采集合规验收

## 1. 实现范围

- 新增来源注册表服务，支持事实层 `fact` 与传播层 `signal` 两类来源。
- 默认内置 `arxiv-ai` 作为国外事实源，`github-trending-ai` 作为传播源。
- 来源字段包含分类、地区、语言、访问模式、授权要求、启停状态、小时限流、刷新间隔、合规说明和最近状态。
- 管理 API 支持列出来源、启停来源和调整小时限流。
- 合规校验拒绝 `bypass` 访问模式，不提交绕过授权、抓取真实 token 或规避平台限制的采集逻辑。

## 2. API 影响

- `GET /api/v1/admin/sources`
- `PATCH /api/v1/admin/sources/{id}`
- `/openapi.json` 已包含来源管理接口。

## 3. 验收命令

```bash
go test ./...
git diff --check
```

## 4. 验收结论

- 至少一组国外事实源和一组传播源已存在。
- 来源可启停、可分类、可配置小时限流。
- 单来源状态通过 `lastStatus` 独立记录，后续采集任务可基于该状态降级，不影响其他来源。
- PostgreSQL 持久化、真实采集适配器和 Redis 任务锁在后续 P0 编号任务中继续实现。
