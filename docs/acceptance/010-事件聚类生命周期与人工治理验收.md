# 事件聚类、生命周期与人工治理验收

## 当前结论

PLAN-010 正在实施，当前结论仍为 `pending`，不得标记为完成或归档。

已落地的可复核范围包括 Schema/领域契约、四通道有界候选召回、确定性决策记录、生命周期与人工治理 API、合并/拆分事务，以及以下补齐项：

- `1b73d00`：候选决策与 Event/成员/Monitor 关系在同一 PostgreSQL 事务中应用；重试和并发重放不重复创建 Event。
- `4b646c0`：active 证据变化事务化重算生命周期、代表成员和 retained 审计；人工锁不被自动覆盖。
- `961f2a2`：固定聚类评测集覆盖跨语言同事件、同实体不同事件、迟到内容、最优候选和灰区复核，并强制 F1 ≥ 75% 与候选上限。
- `261b3df`、`3922caf`：拆分成员按确定顺序加锁并拒绝陈旧成员；合并先解析 canonical target，再校验提交时的目标版本，避免历史 merged target 误报冲突。
- `3dd9acf`：事件 API 将输入错误、缺失资源和版本/状态冲突稳定映射为 400/404/409 numeric code。
- `973b3d8`：以 PostgreSQL trigger 注入成员迁移失败，证明 merge 和 split 都回滚成员、事件版本和治理审计，不留下半迁移关系。
- `4c046e0`、`f1be1e5`：split 和 merge 后都在原事务内重算受影响 Event 的生命周期、代表成员与 retained 审计；源 Event 不再指向已迁出的代表内容。
- `eca4cad`：split 创建的新 Event 从首个迁移成员继承 `content_dedupe_v1` 指纹，保持后续有界候选召回可用。

本地已在可丢弃 PostgreSQL/Redis fixture 上通过相关单元、`-race` 集成和 `make ci`；关键成员迁移故障注入已覆盖。固定评测集当前为 6 个确定性回归样本，不应表述为真实长期标注集。最终验收仍缺少独立复审和真实标注集的长期质量证据；因此 `status`/`result` 保持 `review`/`pending`。

## 验收范围与禁止越界

- 只验收已通过 Monitor 相关性判断的 active Content 到 Event 的聚类、成员、生命周期、合并、拆分、证据重算和人工锁。
- 不验收热度趋势、实体/Claim、AI 摘要、队列/调度、知识库、报告投递或新增 AI task type。
- 合并/拆分只验证 `event_contents`、`monitor_events` 和 `merged_into_id`；实体、Claim、Topic、Report 只验证 `downstream_reconcile_required` 审计事件，不验证其迁移实现。
- 记录准确实现提交 SHA、Schema/record/OpenAPI 变更；不记录真实 DSN、密钥、Provider 原文或本机绝对路径。

## 验收环境与 Fixture

- PostgreSQL/Redis 使用可丢弃 fixture：`HOTKEY_TEST_DSN='postgres:///hotkey_plan010_test?sslmode=disable'`、`HOTKEY_TEST_REDIS_URL='redis://127.0.0.1:6379/15'`。
- 固定数据集版本：`testdata/clustering/v1/evaluation.json`，当前 6 个确定性回归样本（中英文同一事件、同名实体不同事件、迟到内容、最优候选和灰区复核）。它用于防回归，不替代带来源、分组切分和冲突决议的真实标注集。
- 若实现使用 MinIO 原始证据，必须使用专用 disposable fixture 并在验收后明确停止；否则不得声称 MinIO 已验收。

## 必须通过的 GREEN 证据

| 能力 | 证据 |
|---|---|
| 聚类质量与候选上限 | 固定集 F1 ≥ 75%；词法/Monitor ≤16、时间/来源 ≤12、事实指纹 ≤10、同 profile 向量 ≤12，合并去重后每条 Content 精排候选 ≤50；硬冲突不得被向量分数覆盖；无 Event embedding 时有明确规则降级 |
| 决策审计与归属 | `event_clustering_decisions` 保存每个候选的版本、feature_input_hash、通道/排名、五项分数、证据、actor 和 reason；`event_governance_audits` 保存前后状态、source/target、expected_version；非 duplicate Content 只有一个当前 Event 归属 |
| 幂等与稳定键 | 相同输入重跑不创建重复 Event/成员；标题、摘要、成员变化不改变 event_key；独立证据仍保留 |
| 生命周期与证据 | detected/active/cooling/closed/archived/rejected/merged 迁移、迟到、撤回和删除重算均有状态/原因/版本记录 |
| 人工治理 | 事件字段和成员 `manual_locked` 在自动重算、重试和并发写入后保持有效；人工拒绝不会被重复建议 |
| 合并/拆分 | canonical root、防环、目标已 merged、重复关系、人工锁、失败重试和 `downstream_reconcile_required` 均有证据；成员、Monitor 关系、审计、merged_into_id 在单事务内迁移，故障注入回滚后不存在半迁移关系 |
| API 与安全 | `GET /api/v1/events`、`GET /api/v1/events/:id`、`GET /api/v1/events/:id/contents` 为 viewer；`POST /api/v1/events/:id/contents/:content_id/lock` 为 editor，支持 `locked`/解锁；生命周期/合并/拆分为 admin；merge 校验源/目标版本，split 校验源事件和逐成员版本；陈旧目标、陈旧成员、锁定成员和并发操作均返回 409 且无部分写入；错误有 400/404/409 numeric code，不泄露正文、Prompt、密钥、Provider 原文或 SQL |
