# 17-计费套餐额度与用量统计验收

## 验收范围

- 租户套餐分配：`POST /api/v1/admin/tenants/{id}/billing/plan`。
- 租户用量查询：`GET /api/v1/admin/tenants/{id}/billing/usage`。
- 租户用量记录：`POST /api/v1/admin/tenants/{id}/billing/usage`。
- 套餐额度限制采集、刷新和 AI 调用：
  - `collection`
  - `refresh`
  - `ai_call`
- 超出额度时返回 `402`，并返回 `quota_exceeded` 原因。

## 非目标

- 不接入真实支付网关或发票系统。
- 不实现自动扣费、退款或税务流程。
- 不引入 P2 后续复杂消息队列和 Worker 池能力。

## 验收命令

```bash
go test ./...
git diff --check
find docs/product/prd docs/plans -maxdepth 1 -type f | sort -V
```

## 验收结果

- 计费服务测试覆盖按租户统计用量。
- 计费服务测试覆盖采集、刷新和 AI 调用额度限制。
- HTTP 测试覆盖套餐分配、用量记录、超限拦截和用量汇总。
- OpenAPI 测试覆盖计费 API 路径和 `402` 超限响应。
