# 22-传播路径与事实冲突仲裁验收

## 验收范围

- 传播路径写入：`POST /api/v1/admin/events/{id}/propagation`。
- 传播路径查询：`GET /api/v1/events/{id}/propagation`。
- 仲裁 claim 写入：`POST /api/v1/admin/events/{id}/claims`。
- 仲裁结果查询：`GET /api/v1/events/{id}/arbitration`。
- 传播路径按观察时间排序。
- 传播步骤区分事实源和传播源。
- 多事实源同一 claim 出现不同值时返回冲突状态、获胜值、获胜来源和解释。

## 非目标

- 不在本任务实现完整传播图推理、图数据库查询语言或自动事实裁决模型。
- 不绕过来源授权，不接入无授权全平台抓取。
- 不把人工审核工作流、申诉流和复杂风控策略纳入本任务。

## 验收结果

- 服务层测试覆盖传播路径按观察时间排序。
- 服务层测试覆盖事实源和传播源步骤区分。
- 服务层测试覆盖多事实源 claim 冲突时返回仲裁状态和解释。
- HTTP 测试覆盖传播路径写入、查询、claim 写入和仲裁结果查询。
- OpenAPI 测试覆盖传播路径和仲裁 API。
