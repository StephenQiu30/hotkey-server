# 5-内容标准化与去重验收

## 1. 实现范围

- 新增 SourceItem 标准化服务，保留来源、原始 URL、canonical URL、标题、摘要、发布时间、抓取时间、内容 hash 和原始元数据。
- 管理 API 支持写入和列出标准化内容。
- 去重规则按顺序覆盖 canonical URL、内容 hash、24 小时标题窗口。
- 重复内容返回已有 SourceItem，避免后续事件聚合重复消费。

## 2. API 影响

- `GET /api/v1/admin/source-items`
- `POST /api/v1/admin/source-items`
- `/openapi.json` 已包含 SourceItem 管理接口。

## 3. 验收命令

```bash
go test ./...
git diff --check
find docs/product/prd docs/plans -maxdepth 1 -type f | sort -V
```

## 4. 验收结论

- 原始 URL、来源、发布时间、抓取时间、hash 和原始元数据可追踪。
- URL、hash、标题时间窗口去重均有单元测试覆盖。
- HTTP API 的写入、重复写入和列表读取均有测试覆盖。
- PostgreSQL 持久化和事件聚合消费会在后续 P0 编号任务中继续实现。
