## 需求映射

| 领域 | 已复核证据 |
|---|---|
| 邮箱验证与密码 | bcrypt 的 UTF-8/72-byte 边界、验证码/验证票据单次消费、注册和重置、错误凭据不可枚举；SMTP 或 Redis 故障以 `90001`/503 失败且 SMTP 失败不留下验证码状态。 |
| JWT 与可撤销会话 | JWT 仅接受 HS256 并校验 issuer、audience 与时间；每次 Bearer 请求读取当前 PostgreSQL 用户与会话事实，退出、改密、禁用、软删除和重放都会使既有 Access Token 失效。 |
| 轮换、并发与重放 | PostgreSQL `FOR UPDATE` 消费 Refresh Token；并发刷新只有一个成功，旧 Token 重放撤销同一逻辑会话及所有仍有效 Refresh Token。会话绝对上限不可修改，子 Token 不能超过它。 |
| RBAC 与用户生命周期 | `admin` 路由受认证与角色中间件保护；最后 active admin 保护、停用、软删除、重注册、恢复冲突与审计均由 application/PostgreSQL 集成测试覆盖。恢复用户保持 disabled。 |
| Cookie 与 CORS | `hotkey_refresh` 固定为 `HttpOnly`、`SameSite=Strict`、`Path=/api/v1/auth`；生产配置要求 `Secure`。携带 Cookie 的 refresh/logout 只接受显式允许 Origin，认证 CORS 不允许 `*`。 |
| HTTP、OpenAPI 与敏感信息 | `/api/v1/auth` 与受限 `/api/v1/users` 通过 Result 输出；生成 Swagger 受架构测试约束。DTO、日志和审计测试拒绝暴露密码、密码哈希、验证码、验证票据、Refresh Token/Cookie 或 SMTP 内部原因。 |
