# 模块化单体、Schema基线与工程门禁验收

## 结论

验收通过。提交 `26820fcf0d209492dc3c500d205881ecf06e8603` 建立 `cmd/hotkey` 入口、all/api/worker 生命周期、完整单一 Schema、48 张应用表记录映射、5 张 River 基础设施表、Repository 接口契约与 POSIX 门禁。

## 环境与基线

- macOS arm64、Go 1.26.3、PostgreSQL 18.4（`pg_trgm` 与 `vector`）
- 使用并删除临时 `hotkey_schema_test` 数据库；未触及开发或生产库。
- 红灯基线：缺少 `cmd/hotkey`、`internal/modules`、完整 `db/schema.sql`，且旧分片 Schema 不能覆盖设计表和关键约束。

## 绿灯证据

| 验收项 | 命令 | 结果 |
|---|---|---|
| 生命周期稳定性 | `GIN_MODE=release go test ./internal/bootstrap -count=20` | 通过：角色、失败 readyz、端口竞争、失败回滚、取消和重启 |
| 全量回归 | `go test ./... -count=1` 与 `go vet ./...` | 通过 |
| 架构边界 | `make validate` 与 `git diff --check` | 通过：无 split Schema、Migration、AutoMigrate 或 PowerShell-only 门禁 |
| 构建清理 | `make build && make clean` | 通过：二进制已清除 |
| 数据库 | `HOTKEY_TEST_DSN=postgres:///hotkey_schema_test?sslmode=disable make ci` | 通过：Schema 双跑、48 应用表、5 个指定 River 表，以及非法分数/状态、外键、唯一键、幂等键拒绝 |

## 边界与残余风险

- `/healthz`、`/readyz` 使用安全 Result，但为非 `/api/v1` 运维探针；公共 OpenAPI 生成与排除断言由 PRD/PLAN-003 交付。
- pgx/GORM、事务、分页、错误映射、具体 Repository 和数据库命令属于 PRD/PLAN-002。
- 设计 001–004 仍被后续任务依赖，保留 `accepted`；只归档本次已完成的 PRD-001 与 PLAN-001。
