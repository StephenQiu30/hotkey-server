# X 热点监控平台验收

## 验收环境

- Docker Desktop 4.x+
- Go 1.22+
- 8GB+ RAM

## 验收步骤

### 1. 启动本地环境

```bash
bash scripts/start-local.sh
```

**预期结果：**
- postgres、redis、api、worker 四个服务启动成功
- `docker compose ps` 显示所有服务 healthy

### 2. 验证健康检查

```bash
curl http://localhost:8080/healthz
```

**预期结果：**
```json
{"status":"ok"}
```

### 3. 验证结构化日志

```bash
docker compose logs api | head -5
```

**预期结果：**
- 日志包含 `"service":"api"` 字段
- 日志格式为 JSON

### 4. 验证 Worker 启动

```bash
docker compose logs worker | head -5
```

**预期结果：**
- 日志包含 `"service":"worker"` 字段
- 日志显示 `starting` 和 `ready` 消息

### 5. 运行验证脚本

```bash
bash scripts/validate-repository.sh
```

**预期结果：**
- 所有检查通过
- Go 测试全部通过
- Docker Compose 配置有效
- 无错误输出

### 6. 运行 Go 测试

```bash
go test ./...
```

**预期结果：**
- 所有包测试通过
- observability 包包含 logger、metrics、health 测试

### 7. 创建监控任务（联调）

1. 通过 API 注册用户
2. 登录获取 token
3. 创建监控任务
4. 配置关键词和采集频率

**预期结果：**
- 任务创建成功
- 任务状态显示为"待执行"

### 8. 触发采集

1. 等待定时任务触发
2. 查看 worker 日志

**预期结果：**
- 采集任务执行成功
- 日志显示采集结果

### 9. 查看内容流

1. 调用内容流 API
2. 查看采集到的内容

**预期结果：**
- 内容列表正常返回
- 内容详情可查看

### 10. 查看主题和趋势

1. 调用主题分析 API
2. 查看热点主题和趋势数据

**预期结果：**
- 主题列表正常返回
- 趋势数据正确

### 11. 触发提醒

1. 配置提醒规则
2. 等待触发条件满足

**预期结果：**
- 提醒正常触发
- 通知发送成功

## 验收清单

- [ ] 本地可以一键起服务（API + Worker + DB + Redis）
- [ ] 健康检查端点 `/healthz` 返回 200
- [ ] 结构化日志包含 `service` 字段
- [ ] 计数器指标可递增和读取
- [ ] 验证脚本覆盖后端和 compose 配置
- [ ] 验收文档能指导完整联调

## 已知限制

- 首版不包含生产级监控
- 仅支持本地开发环境
- 不包含多环境部署矩阵
- 不包含 Web 前端部署（纯后端基线）
