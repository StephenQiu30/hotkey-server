# X 热点监控平台验收

## 验收环境

- Docker Desktop 4.x+
- Go 1.22+
- Node.js 20+
- 8GB+ RAM

## 验收步骤

### 1. 启动本地环境

```bash
bash scripts/start-local.sh
```

**预期结果：**
- 所有服务启动成功
- API 健康检查通过：`curl http://localhost:8080/health`

### 2. 验证健康检查

```bash
curl http://localhost:8080/health
```

**预期结果：**
```json
{"status":"ok"}
```

### 3. 验证结构化日志

```bash
docker compose logs api | head -5
```

**预期结果：**
- 日志包含 `"service":"api"` 字段
- 日志格式为 JSON

### 4. 运行验证脚本

```bash
bash scripts/validate-repository.sh
```

**预期结果：**
- 所有检查通过
- 无错误输出

### 5. 创建监控任务

1. 打开 http://localhost:3000
2. 登录系统
3. 创建新的监控任务
4. 配置关键词和采集频率

**预期结果：**
- 任务创建成功
- 任务状态显示为"待执行"

### 6. 触发采集

1. 等待定时任务触发，或手动触发采集
2. 查看采集日志

**预期结果：**
- 采集任务执行成功
- 日志显示采集结果

### 7. 查看内容流

1. 进入内容流页面
2. 查看采集到的内容

**预期结果：**
- 内容列表正常显示
- 内容详情可查看

### 8. 查看主题和趋势

1. 进入主题分析页面
2. 查看热点主题和趋势图

**预期结果：**
- 主题列表正常显示
- 趋势图数据正确

### 9. 触发提醒

1. 配置提醒规则
2. 等待触发条件满足

**预期结果：**
- 提醒正常触发
- 通知发送成功

## 验收清单

- [ ] 本地可以一键起服务
- [ ] 健康检查端点 `/health` 返回 200
- [ ] 结构化日志包含 service 字段
- [ ] 计数器指标可递增和读取
- [ ] 验证脚本覆盖后端、前端和 compose 配置
- [ ] 验收文档能指导完整联调

## 已知限制

- 首版不包含生产级监控
- 仅支持本地开发环境
- 不包含多环境部署矩阵
