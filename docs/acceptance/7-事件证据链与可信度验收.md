# 7-事件证据链与可信度验收

## 1. 实现范围

- 新增事件证据链服务，按 `fact` 和 `signal` 分离事实证据与传播证据。
- 事实证据按可信等级贡献事实分，传播证据只贡献热度分。
- 低可信传播源不能生成事实证据或事实分。
- 事件详情返回事实证据、传播证据、事实分、热度分、可信标签、风险说明和 AI 总结。
- AI 总结必须携带来源引用，缺少 citation 时拒绝写入。

## 2. API 影响

- `POST /api/v1/admin/event-evidence`
- `POST /api/v1/admin/events/{id}/ai-summary`
- `GET /api/v1/events/{id}/evidence`
- `/openapi.json` 已包含事件证据链接口。

## 3. 验收命令

```bash
go test ./...
git diff --check
find docs/product/prd docs/plans -maxdepth 1 -type f | sort -V
```

## 4. 验收结论

- 低可信传播源只能贡献热度，不会污染事实证据。
- 事件详情可看到事实证据与传播证据。
- AI 总结强制携带来源引用。
- 当前实现先使用进程内仓储锁定领域和 OpenAPI 契约；后续数据库任务会接入持久化和真实事件详情读写。
