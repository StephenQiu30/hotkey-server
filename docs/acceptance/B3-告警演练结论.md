## 指标基线

接口：`GET /api/ops/metrics`（需登录 token）。

演练后指标：

```json
{
  "total_requests": 139,
  "status_buckets": {"200": 122, "201": 4, "401": 1, "429": 12},
  "status_by_class": {"2xx": 126, "3xx": 0, "4xx": 13, "5xx": 0, "unknown": 0},
  "active_rate_limit_clients": 3,
  "rate_limit_exceeded_total": 12
}
```

## 告警策略

| 指标 | 阈值建议 | 等级 | 处理 |
|---|---|---|---|
| `rate_limit_exceeded_total` | 5 分钟内新增 > 10 | P2 | 检查异常客户端、必要时临时调高/调低限流 |
| `status_by_class.5xx` | 5 分钟内新增 > 0 | P1 | 检查 API 日志与数据库连接 |
| check run failure | 单次 `failure_count > 0` | P2 | 查看 `error_summary`，区分凭据缺失、外部超时、通知失败 |
| notification failed/skipped | 生产 SMTP 场景出现 failed 或连续 skipped | P2 | 检查 SMTP 配置与发件策略 |
| AI fallback | 生产模型配置下连续 fallback | P2 | 检查模型网关、key、base URL 与配额 |
