# AI 热点监控工具验收标准

本验收标准以当前仓库实现为主线：先满足 MVP 运行闭环，再分步推进企业级补齐。
如果目标是企业级 P0 能力，则按 `openspec/changes/enterprise-p0-hotspot-platform/tasks.md` 的任务验收。

## 适用边界（2026）

- 本阶段以 `docs/product/产品需求文档.md` 的 `MVP` 范围为准。
- `docs/product/prd/00-企业级AI热点监控平台PRD.md` 为企业级扩展方向，仅在明确启动企业级 P0 时执行。
- 不验收旧实现兼容性、旧数据库迁移兼容和旧 OpenAPI 契约。

## 企业级 P0 追加验收（可逐步收口）

- [ ] A1. Provider 注册与可插拔链路完整且有最少 2 条来源接入实现。
- [ ] A2. LLM Provider 抽象可切换，主模型异常时有降级/跳过策略且有可观察日志。
- [ ] A3. 核心抓取与 AI 分析链路失败隔离；单源失败不影响全量执行。
- [ ] A4. 热点聚类、去重、版本化元数据可检索回溯。
- [ ] A5. RSS 端点可提供至少：关键词/用户/热门（keyword/user/trending）三类入口。
- [ ] A6. 报告系统支持 Markdown 输出，HTML 预览路径可用，发送失败记录可读。
- [ ] A7. 日志与监控具备可检索的抓取、AI、邮件、任务指标。
- [ ] A8. 安全增强项（JWT、RBAC、限流）按企业级任务分批验收，不要求本阶段全部上线。

## 1. 已通过自动化或静态确认

- [x] FastAPI 后端入口位于 `apps/api`。
- [x] Next.js 前端位于 `apps/web`（当前页面入口按项目约定可访问）。
- [x] PostgreSQL 是当前 P0 唯一数据库。
- [x] `sql/001_init_schema.sql` 是 schema 事实源，包含 `keywords`、`sources`、`hotspots`、`ai_analyses`、`notifications`、`check_runs`、`reports`、`settings`。
- [x] SQLAlchemy models 与 `sql/001_init_schema.sql` 保持核心表一致。
- [x] RSS、Hacker News、X/Twitter、Bing、Bilibili、Sogou-style 具备 adapter 路径。
- [x] X/Twitter 未配置 Token 时跳过来源并不中断流程。
- [x] Bing 未配置 Key 时跳过来源并不中断流程。
- [x] AI 查询扩展可在本地降级路径生成 2-5 个去重查询词。
- [x] AI 分析结果包含真实性、相关性分数、相关性理由、关键词命中、重要性和摘要。
- [x] `active`/`filtered` 阈值过滤规则已实现：`relevance_score >= RELEVANCE_THRESHOLD` 且 `is_real is not False` 才进入 `active`。
- [x] `filtered` 热点不发送事件邮件、不进入日报/周报。
- [x] SMTP 缺失时系统仍可运行，并记录 `skipped`。
- [x] `/api/search` 可执行即时搜索，搜索结果不入库。
- [x] `/api/reports` 可生成 daily/weekly 报告，并只聚合 `active` 热点。
- [x] `/api/daily-reports` 不再注册，已移除兼容别名。
- [x] `/rss/trending`、`/rss/keyword/{keyword_name}`、`/rss/user/{user_id}` 可访问并返回 XML。
- [x] 前端提供核心页面：关键词、来源、热点、搜索、任务、报告、通知和设置。

## 2. 需真实配置后验收

- [ ] 空 PostgreSQL 通过 `sql/001_init_schema.sql` 初始化并完成一次端到端检查。
- [ ] 使用真实 X API v2 Recent Search Token 验证 X/Twitter 来源。
- [ ] 使用真实 Bing Search Key 验证 Bing 来源。
- [ ] 使用真实 OpenAI 兼容模型配置验证查询扩展和热点分析。
- [ ] 使用真实 SMTP 配置验证事件邮件和报告邮件发送。
- [ ] 后端与 PostgreSQL 同时启动后，使用浏览器验证工作台创建关键词、搜索、报告生成与发送链路。

## 3. 人工功能验收

- [ ] 可以创建、更新、删除、启停关键词。
- [ ] 同一 URL + 来源组合不能重复入库。
- [ ] 热点可按关键词、来源、重要性、时间范围筛选。
- [ ] 热点可按时间、相关性或重要性排序。
- [ ] 可手动触发热点检查。
- [ ] 可简单定时触发热点检查。
- [ ] 报告内容以本地 Markdown 模板生成；AI 报告增强不作为当前 P0 必需。

## 4. 非目标确认（当前阶段）

- [x] P0 不引入多租户。
- [x] P0 不引入复杂权限和分级计费。
- [x] P0 不引入实时推送。
- [x] P0 不引入向量库。
- [x] P0 不强制依赖 Celery/Redis。
- [x] P0 不兼容旧表结构、旧内存仓库或旧 bootstrap 示例数据。
