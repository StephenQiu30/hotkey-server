# Plan 文档

本目录存放执行计划类文档。

## 适合放入

1. 阶段计划与任务拆解。
2. 任务编排和归档路线。
3. 里程碑、排期和交付顺序。
4. 风险、依赖和后续事项。

## 不适合放入

1. 产品需求原文。
2. 详细技术设计。
3. 验收报告或测试记录。
4. 仅服务当前执行的 todo 列表、临时进展记录或过程流水账。

## 命名建议

使用 `序号-主题-实现计划.md`，编号全局连续递增，例如 `13-配置与环境启动实现计划.md`。

Plan 必须包含 `docs/TEMPLATE.md` 规定的 frontmatter 与必含章节：目标、文件清单、任务拆解、TDD 与验证、执行顺序、回滚策略、验收命令、变更记录。
