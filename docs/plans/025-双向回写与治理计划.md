# Obsidian Writeback Governance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立结构化白名单回写、冲突检测、审计记录和回写后再导出一致性机制，保证知识中台闭环可控。

**Architecture:** `internal/obsidian` 负责解析白名单字段，`internal/knowledge` 负责校验、冲突比较和回写命令，`internal/jobs` 负责批处理，`internal/database` 负责审计与状态落库。人工知识只允许落到 `event_annotations`、`topic_annotations`、`theme_memberships` 等 sidecar 表，自由文本永不作为默认回写输入。

**Tech Stack:** Go, PostgreSQL, GORM, Markdown frontmatter parser, worker jobs, integration tests

```go
func DetectConflict(in ConflictInput) error {
    if in.CurrentRevision != "" && in.IncomingRevision != in.CurrentRevision {
        return ErrWritebackConflict
    }
    return nil
}
```

- [ ] **Step 4: 实现回写应用服务，仅写入人工知识 sidecar**

```go
func (s *Service) ApplyChange(ctx context.Context, change WritebackChange) error {
    if err := ValidateWriteback(change); err != nil {
        return err
    }
    if err := DetectConflict(change.ConflictInput); err != nil {
        return err
    }
    return s.sidecarRepo.Apply(ctx, change)
}
```
