# 身份认证会话与权限实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 交付以 PostgreSQL 为事实源、可撤销且可审计的邮箱密码身份能力，并让后续模块可复用认证主体与角色检查。

**Architecture:** `identity` 以 domain/application/infrastructure/transport/http 四层拥有用户、逻辑会话和单用途 Refresh Token；平台 HTTP 只依赖一个窄认证端口。Redis 仅用于邮箱验证码、验证票据、频率和短锁；bcrypt/JWT/SMTP/Redis 均在 infrastructure，用户和会话正确性始终由 PostgreSQL 事务决定。

**Tech Stack:** Go 1.26.3、Gin 1.12、GORM/pgx v5、PostgreSQL 16+、Redis 8、`github.com/golang-jwt/jwt/v5 v5.3.1`、`github.com/redis/go-redis/v9 v9.18.0`、`golang.org/x/crypto/bcrypt`、标准库 SMTP/TLS、Fx、Swaggo。

## 全局约束

- 只修改 `hotkey-server`；不创建迁移目录、不使用 AutoMigrate、不修改 Web 或 Miniapp。
- `db/schema.sql`、`internal/platform/database/model` 和 schema 架构测试必须同步；对既有数据库不做服务启动时变更，验收仅使用可丢弃空库。
- Access Token 固定 15 分钟；Refresh Token 是 Cookie 内 32 字节以上随机不透明值，7 天滑动、30 天绝对上限，数据库只保存 SHA-256 哈希。
- 每次 Bearer 请求必须确认 JWT HS256/issuer/audience/时间、活动用户和未撤销会话；JWT 不是会话事实。
- 所有模块 HTTP handler 使用 `Wrap` 与 Result，不直接输出 Gin JSON；错误码注册后不可改义。
- Redis 故障不能改变既有登录、刷新、退出、撤销或 Bearer 校验的正确结论；它只使验证码/票据流程返回 `90001`/503。
- 不记录密码、Token、Cookie、验证码、验证票据、bcrypt 哈希或完整邮箱；Cookie 固定 `HttpOnly`、`SameSite=Strict`、`Path=/api/v1/auth`，生产环境 `Secure`。
- 每个任务先留下可复现红灯，再写最小实现并运行局部绿灯；任务边界处只提交当前任务文件。
- 每个任务的绿灯提交后必须由独立 Reviewer 审查其 diff、测试范围、Schema/API 同步和安全边界；关闭全部 P0/P1 后才可开始下一任务，最终验收还要复审整个任务链。

所有后续任务使用以下签名，不创建第二套 Repository 或全局可变认证状态：

```go
type Role string
const (RoleAdmin Role = "admin"; RoleEditor Role = "editor"; RoleViewer Role = "viewer")

type Subject struct { UserID, SessionID int64; Role Role }
type Authenticator interface { Authenticate(context.Context, string) (Subject, error) }
type Service interface {
    Register(context.Context, RegisterInput) (User, error)
    Login(context.Context, LoginInput) (LoginResult, error)
    Refresh(context.Context, string) (LoginResult, error)
    Logout(context.Context, Subject, string) error
    ChangePassword(context.Context, Subject, ChangePasswordInput) error
    ConfirmPasswordReset(context.Context, ResetPasswordInput) error
    ListUsers(context.Context) ([]User, error)
    UpdateUser(context.Context, Subject, int64, UpdateUserInput) (User, error)
    DeleteUser(context.Context, Subject, int64) error
    RestoreUser(context.Context, Subject, int64) (User, error)
}
```
