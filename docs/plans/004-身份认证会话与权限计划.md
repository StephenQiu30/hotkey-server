# 身份认证会话与权限计划

## 计划目标

交付注册、登录、刷新、退出、密码修改、用户停用和 admin/editor/viewer 授权闭环。

## 开工条件

- 对应 PRD 的 status 为 accepted，execution_status 为 ready
- frontmatter 中 depends_on 列出的 Plan 全部为 done
- main 已同步，工作区只包含当前任务相关文件

## 执行文件

| 动作 | 路径 | 目的 |
|---|---|---|
| 创建 | internal/modules/identity/domain/*.go | User、Session、Role 与 Repository |
| 创建 | internal/modules/identity/application/*.go | 注册、登录、刷新、退出和管理用例 |
| 创建 | internal/modules/identity/infrastructure/postgres/*.go | GORM Repository |
| 创建 | internal/modules/identity/infrastructure/security/*.go | bcrypt 与 JWT 适配 |
| 创建 | internal/modules/identity/transport/http/*.go | DTO、Handler 与路由 |
| 修改 | db/schema.sql | users、preferences、sessions |
| 修改 | internal/bootstrap/app.go | identity 装配 |
| 修改 | internal/platform/http/router.go | auth 与 users 路由 |
| 创建 | internal/modules/identity/**/*_test.go | 领域、应用与集成测试 |
| 修改 | OpenAPI 生成注解与错误码 | 公共契约 |

## 执行步骤

1. 先写密码、会话轮换、撤销、RBAC 和并发刷新红灯测试。
2. 同步 Schema 与 identity 记录模型。
3. 实现领域规则与 PostgreSQL Repository。
4. 实现 bcrypt、JWT、会话轮换和撤销。
5. 实现应用权限和统一 Result HTTP 接口。
6. 运行真实 PostgreSQL 集成测试及无 Redis 模式验证。

## 验收命令

| 阶段 | 命令 | 通过标准 |

| 红灯 | go test ./internal/modules/identity/... -count=1 | 新用例因实现缺失失败 |

| 集成 | go test -tags=integration ./internal/modules/identity/... -count=1 | PostgreSQL 会话链路通过 |
| 契约 | make openapi && make openapi-validate | auth/users 契约通过 |
| 全量 | make ci | 全部通过 |

## 验收清单

- 邮箱规范化唯一，密码仅保存 bcrypt 哈希
- 并发刷新只产生一个新会话结果
- 退出、改密和停用撤销相关会话
- 三种角色权限满足 PRD
- 无 Redis 时认证正确性不变
- 日志和 API 不泄露密码、Token、会话密钥或内部错误

## 提交边界

- test: 定义身份与会话行为
- impl: 实现 identity 领域和持久化
- feat: 发布 auth 与 users 安全 API


## 风险与回滚

- 实现需要改变 Design 或 PRD 契约时立即停止，先更新正式文档并重新复核
- 下游 Plan 开工前，可整体回退本任务提交并同步恢复 Schema、OpenAPI 和测试
- 下游 Plan 已开工后，使用新的前向修复 Plan，不恢复旧双轨或兼容实现
