## 目的

把企业级 P0 的后端能力补齐与运营级验收统一为一份可执行清单：

- 最后按依赖关系批次执行。

## 二、批次编排（一次性执行顺序）

### 第 1 批（启动条件）

- [x] **并行预检**执行 D0（数据库兼容与可复测闭环）

- A 阶段已闭环（A2/A4/A8）
- 为 B/C 阶段预留环境与证据目录

### 第 2 批（并行启动）

- [x] **并行**执行 B1、B2（可互不阻塞）
- 产出：`docs/acceptance/B1-回放证据包.md` 与 `docs/acceptance/B2-健康可达性问题清单.md`

### 第 3 批（串行衔接）

- [x] **在 B1/B2 完成且证据齐备后**启动 B3
- 产出：`docs/acceptance/B3-告警演练结论.md`

### 第 4 批（并行收口）

- [x] 与 B3 结论联动并行推进 C1、C2
- 产出：`docs/acceptance/C1-运维手册补充.md`、`docs/acceptance/C2-阈值与变更SOP.md`

## 三、任务验收边界（冻结条件）

- Issue 关闭前需补齐：
  - 最小复现步骤或命令；
  - 证据文件（截图、日志、时间轴）；
  - 失败场景的降级说明；
  - 与父 Issue 的闭环引用。
