# X Hotspot Monitoring Platform Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在空壳仓库中搭建一个可上线的 X 热点监控平台 MVP，覆盖用户监控、内容采集、主题趋势、提醒通知和管理后台。

**Architecture:** 采用 Go API + Go Worker + PostgreSQL + Redis 的分层单体架构。同步接口和异步任务链路分离，所有采集和分析能力通过平台接入层和任务层解耦。Web 前端由独立仓库 `hotkey-web` 承担。

**Tech Stack:** Go, PostgreSQL, Redis, Docker, SQL migration, X API

---

## 背景

当前仓库只有治理与文档骨架，没有业务实现。实施时必须先建立统一目录和验证方式，再逐步实现平台能力，避免不同子模块采用各自风格导致后续重构。

## 范围

本计划只负责实施顺序与依赖，不替代各子计划中的细粒度任务。具体编码步骤见 `002` 到 `007` 号计划。

## 目标

1. 明确每一阶段的前置条件和交付物。
2. 保证每一阶段结束后都能通过明确验证命令。
3. 保证先落基础设施和数据契约，再落采集、分析、通知和前端。

## 非目标

1. 不在本计划中写每一个 handler 或 SQL 明细。
2. 不在本计划中定义页面视觉稿。
