# Alerts And Notification Delivery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在主题和趋势结果基础上生成提醒事件，并将事件投递为站内通知和邮件记录。

**Architecture:** `internal/alert` 只负责规则判定和事件生成，`internal/notify` 负责用户通知与投递，不让邮件发送细节污染提醒规则判断。

**Tech Stack:** Go, PostgreSQL, SMTP or provider SDK, fake mailer tests

## 验收门禁

1. 趋势信号可以触发提醒事件。
2. 用户可以查询和已读通知。
3. 邮件投递结果有审计记录。

## 风险与边界

1. 首版邮件模板保持极简文本或基础 HTML。
2. 若邮件服务未定，先用 fake mailer 和本地 SMTP 捕获服务验证。
