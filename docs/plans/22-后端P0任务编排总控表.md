## 二、执行顺序（推荐批次）

1. **第一波：并行执行 B1、B2、B3 的并行预热**
   - 前置：环境凭据可达，数据库初始化成功。
2. **第二波：B1 与 B2 完成后进入 B3（告警闭环）**
   - 前置：指标基线与故障注入场景可复现。
3. **第三波：C1 与 C2 并行收口**
   - 前置：B2/B3 的关键证据链已提交。

## 三、验收边界（本轮）

- A 阶段不再开新功能 PR，视为已闭环。
- 本轮待补齐只保留：
  - 真实环境回放与凭据演练（B1）
  - 部署健康可达与可恢复性（B2）
  - 告警策略到演练闭环（B3）
  - 运维文档化与阈值流程化（C1/C2）
- 任何新增需求必须先补 PRD 再进入 PLAN，再拆分 Issue，避免“无归属任务”再次出现。
