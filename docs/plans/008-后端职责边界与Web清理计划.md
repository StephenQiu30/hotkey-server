# Backend Boundary Cleanup Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 将 `hotkey-server` 收敛为纯后端服务，清理所有 Web 前端相关规划、目录、验证命令和任务引用，仅保留业务 API 与管理运维 API。

**Architecture:** 后端仅保留 Go API + Go Worker + PostgreSQL + Redis，Web 前端职责迁移至独立的 `hotkey-web` 仓库。

**Tech Stack:** Go, PostgreSQL, Redis, Docker

## 验收门禁

1. `hotkey-server` 文档不再承担 Web 前端实现职责。
2. 仓库内不再存在 `web/`、`Next.js`、前台页面和后台页面实现计划。
3. 管理运维 API 仍然保留在后端设计范围内。
4. Linear 执行栈与新的纯后端边界一致。

## 风险与边界

1. 本计划仅清理文档引用，不涉及代码或配置的实际删除。
2. 外部 `hotkey-web` 仓库不受影响。
