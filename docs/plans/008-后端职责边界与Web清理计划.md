# Backend Scope Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 `hotkey-server` 明确收敛为纯后端服务，清理所有 Web 前端相关规划、目录、验证命令和任务引用，仅保留业务 API 与管理运维 API。

**Architecture:** 通过一轮文档、计划、验证和任务源清理，去掉 `web/`、`Next.js`、前台页面、管理后台页面等职责表述，同时保留 server 侧认证、监控、分析、通知与管理运维接口设计。

**Tech Stack:** Markdown docs, Linear, GitHub, shell validation

## 验收门禁

1. `hotkey-server` 文档不再承担 Web 前端实现职责。
2. 仓库内不再存在 `web/`、`Next.js`、前台页面和后台页面实现计划。
3. 管理运维 API 仍然保留在后端设计范围内。
4. Linear 执行栈与新的纯后端边界一致。

## 风险与边界

1. 外部 `hotkey-web` 仓库如仍需消费 API，不应在本仓重新承载其页面计划。
2. 清理过程中要保留“server 是 OpenAPI/API 事实源”这条边界，不能把管理能力一起删掉。
