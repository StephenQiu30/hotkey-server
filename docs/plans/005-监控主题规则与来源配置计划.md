# 监控主题规则与来源配置计划

## 计划目标

交付 Monitor 草稿、预览、发布、暂停、归档及来源凭据引用的完整配置闭环。

## 开工条件

- 对应 PRD 的 status 为 accepted，execution_status 为 ready
- frontmatter 中 depends_on 列出的 Plan 全部为 done
- main 已同步，工作区只包含当前任务相关文件

## 执行文件

| 动作 | 路径 | 目的 |
|---|---|---|
| 创建 | internal/modules/monitor/domain/*.go | Monitor、Rule、状态与 Repository |
| 创建 | internal/modules/monitor/application/*.go | 创建、发布、预览、暂停和归档 |
| 创建 | internal/modules/monitor/infrastructure/postgres/*.go | Monitor 持久化 |
| 创建 | internal/modules/monitor/transport/http/*.go | Monitor API |
| 创建 | internal/modules/source/domain/connection.go | SourceConnection 与凭据引用 |
| 创建 | internal/modules/source/application/connection_service.go | 来源配置用例 |
| 创建 | internal/modules/source/infrastructure/postgres/*.go | 来源持久化 |
| 创建 | internal/modules/source/transport/http/*.go | Source 配置 API |
| 修改 | db/schema.sql | monitors、rules、monitor_sources、connections |
| 修改 | internal/bootstrap/app.go、internal/platform/http/router.go | 模块与路由装配 |
| 创建 | internal/modules/monitor/**/*_test.go、internal/modules/source/**/*_test.go | 状态、权限和预览测试 |

## 执行步骤

1. 先写状态机、乐观锁、权限和预览无副作用测试。
2. 同步完整 Schema 和记录模型。
3. 实现 Monitor 聚合、发布版本和来源关联。
4. 实现 SourceConnection 凭据引用与脱敏查询。
5. 实现预览读取端口，禁止写正式匹配、Event 或 Job。
6. 发布安全 API、OpenAPI 和审计事件。

## 验收命令

| 阶段 | 命令 | 通过标准 |
|---|---|---|
| 红灯 | go test ./internal/modules/monitor/... ./internal/modules/source/... -count=1 | 状态与权限测试失败 |
| 绿灯 | go test ./internal/modules/monitor/... ./internal/modules/source/... -count=1 | 全部通过 |
| 集成 | go test -tags=integration ./internal/modules/monitor/... ./internal/modules/source/... | PostgreSQL 事务通过 |
| 契约 | make openapi && make openapi-validate | monitors/sources 契约通过 |
| 全量 | make ci | 全部通过 |

## 验收清单

- 并发发布只有一个版本成功
- paused 和 archived 不进入到期查询
- 预览不写正式业务事实
- AI 候选词未 approved 时不进入发布版本
- 来源密钥不出现在日志和响应
- 角色权限与审计完整

## 提交边界

- test: 定义 Monitor 配置与发布行为
- impl: 实现 monitor/source 配置模块
- feat: 发布配置与预览 API


## 风险与回滚

- 实现需要改变 Design 或 PRD 契约时立即停止，先更新正式文档并重新复核
- 下游 Plan 开工前，可整体回退本任务提交并同步恢复 Schema、OpenAPI 和测试
- 下游 Plan 已开工后，使用新的前向修复 Plan，不恢复旧双轨或兼容实现
