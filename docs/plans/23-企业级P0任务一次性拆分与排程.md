## 三、执行顺序（一次性排程）

1. **第一段（并发）**：B1 与 B2 预热（依赖 A 阶段闭环）。
2. **第二段（串行）**：B1 与 B2 证据齐备后，立即执行 B3 演练。
3. **第三段（并行收口）**：C1 与 C2 与 B3 结果联动，更新运维手册/阈值流程。
4. 所有阶段完成后更新：`docs/enterprise-p0-backlog-gap-review.md` 与 `docs/验收差距清单.md`。

## 四、验收口径（统一）

- 任何 Issue 关闭前，必须在关闭前补齐：
  - 最小复现步骤
  - 命令/日志/截图（或 API 回应样例）
  - 失败时的降级处理说明
  - 与父 Issue 的闭环引用
