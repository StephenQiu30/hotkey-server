# 29-n8n日报候选与日报保存实现计划

## 1. 目标

让 n8n 可以从后端获取日报候选，并把 AI 整理后的 Markdown、HTML 和结构化 JSON 保存回后端。

## 2. 文件清单

- `internal/report/service.go`
- `internal/httpapi/router.go`
- `internal/httpapi/*_test.go`
- `internal/openapi/spec.go`
- `db/schema.sql`

## 3. 任务拆解

- 实现 `GET /api/v1/internal/reports/daily-candidates`。
- 候选返回 eventId、title、summary、keywords、heatScore、trustScore、factEvidence、signalEvidence、sourceUrls。
- 实现 `POST /api/v1/internal/reports/daily` 保存日报内容。
- 保存字段包含 reportDate、tenantId、markdown、html、structuredPayload、workflowName、executionId。
- 日报保存成功后返回 reportId，供 n8n 邮件状态回写。

## 4. TDD 与验证

- 候选为空时返回空列表而不是错误。
- 候选必须包含可回溯来源链接。
- 保存日报必须校验 reportDate、tenantId 和正文内容。
- 同一日期同一租户幂等保存不会产生重复日报。

## 5. 执行顺序

1. 先补候选接口测试。
2. 再补保存接口测试。
3. 实现 report service 和 HTTP handler。
4. 同步 OpenAPI。

## 6. 回滚策略

- 保留现有 `/api/v1/reports/daily` 查询接口。
- 保存接口失败不影响候选查询。

## 7. 验收命令

```bash
go test ./internal/report ./internal/httpapi ./internal/openapi
go test ./...
```
