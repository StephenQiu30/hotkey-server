# 接入源适配器改造与可替换实现计划（v1）

## 1. 目标

在不影响主链路吞吐与兼容性的前提下，引入来源健康驱动路由，单源失败不中断。

## 2. Issue 映射

- #44 P0: 接入源适配器路由与健康退化

## 3. 任务拆解（含文件）

### T1 选择器与接口
- 新建：`server/app/services/providers/selector.py`
- 修改：`server/app/services/providers/registry.py`

### T2 健康记录
- 新建：`server/app/services/providers/health.py`
- 修改：`server/app/services/ingestion.py`

### T3 执行链路改造
- 修改：`server/app/services/check_runner.py`
- 修改：`server/app/services/search.py`

### T4 观测
- 修改：`server/app/main.py`
- 修改：`server/app/services/ingestion.py`

## 4. TDD 测试清单

- `test_source_route_skips_failing_source`
- `test_source_retry_threshold_triggers_fallback`
- `test_source_fallback_preserves_primary_flow`
- `test_check_runner_still_completes_when_one_source_fail`
- `test_source_rotation_keeps_cluster_and_dedup`

## 5. 交付门禁

- P0 关键链路可用性不下降。
- 失败来源切换日志可追溯。
- 去重与聚类不受路由替换影响。

## 6. 风险与回滚

- 路由策略抖动：需最小窗口和冷却。
- 回退：保留原有 `build_provider` 路径，支持快速切回。
