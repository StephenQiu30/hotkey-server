### Layer 1: 应用组装 + 配置（ADR-003~006）

**Scope:** Cobra + Viper + Fx + Wire；**不改 HTTP 路由实现**

**Files:**
- Create: `cmd/hotkey/main.go`, `api.go`, `worker.go`
- Create: `internal/app/wire.go`, `wire_gen.go`, `api.go`, `worker.go`
- Create: `internal/platform/config/module.go`
- Modify: `internal/config/config.go` → viper
- Modify: `Dockerfile`, `Makefile`, `docker-compose.yml`（入口二进制名）

- [ ] **Step 1:** cobra 注册 `api` / `worker`
- [ ] **Step 2:** viper 加载 `.env`，校验 `DATABASE_URL` / `JWT_SECRET`
- [ ] **Step 3:** wire 生成 Injector；fx 模块注册 config/database/observability
- [ ] **Step 4:** 搬迁现有 main 装配逻辑；入口 < 50 行
- [ ] **Step 5:** 保留 smoke profile（无 DB stub 路径）

**Validation:**

```bash
make test
make validate
go build ./...
```

### Layer 3: Store + Redis + Queue（ADR-010~012）

**Scope:** sqlc + go-redis + asynq 基础设施；**不改业务聚合规则**

**Files:**
- Create: `sqlc.yaml`, `db/queries/*.sql`
- Create: `internal/database/sqlc/`（generated）
- Create: `internal/platform/redis/client.go`
- Create: `internal/platform/queue/asynq.go`
- Modify: `internal/app/worker.go` 注册 asynq server

- [ ] **Step 1:** sqlc 配置 + 首批 query（auth/monitor 已有表）
- [ ] **Step 2:** Repository 逐步改用 sqlc（与手写 SQL 逐域替换）
- [ ] **Step 3:** go-redis 连接 + health check
- [ ] **Step 4:** asynq client/server wiring；enqueue 骨架
- [ ] **Step 5:** docker-compose redis 健康检查通过

**Validation:**

```bash

make validate
```
