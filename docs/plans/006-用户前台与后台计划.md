# Web Dashboard And Admin Console Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 提供创作者可直接使用的前台平台和管理后台，使监控任务、内容、主题、趋势和通知结果可视化。

**Architecture:** 单个 Next.js 工程承载营销页、用户仪表盘和管理后台，通过路由分组 `(marketing)`、`(dashboard)`、`admin` 分区，并统一消费 API 层。

**Tech Stack:** Next.js, TypeScript, React, CSS modules or Tailwind, Playwright or Vitest

## 验收门禁

1. 用户可登录并查看监控任务。
2. 用户可查看内容流、主题和趋势。
3. 用户可查看提醒中心。
4. 管理员可查看任务运行和连接器状态。

## 风险与边界

1. 首版页面先以功能可用为主，视觉系统可在后续单独升级。
2. 若后端 API 尚未完成，可使用 mock 数据先完成前端测试与页面骨架。
