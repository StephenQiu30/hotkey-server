# 04-来源与采集策略实现计划

## 1. 目标

支持 RSS、官方 API、公开页面来源的配置、合规策略和采集运行记录。

## 2. 文件清单

- 创建：`migrations/000004_sources.up.sql`
- 创建：`internal/domain/source/`
- 创建：`internal/service/source/`
- 创建：`internal/platform/fetcher/`
- 创建：`internal/repository/postgres/sourcerepo/`
- 创建：`internal/transport/http/handlers/source/`

## 3. 任务拆解

1. 创建 `sources`、`source_channel_links`、`collection_runs`。
2. 实现来源 CRUD、启停、合规说明校验。
3. 实现 RSS fetcher 和 public page fetcher 接口。
4. 管理员测试采集 API。
5. 记录 collection run 状态和错误。

## 4. TDD 与验证

- 先写创建 public_page source 无 compliance_note 失败测试。
- 写 disabled source 不被采集测试。
- 使用 fake HTTP server 测试 RSS 采集。

## 5. 执行顺序

1. `test:` source validation 和 fake fetcher 测试。
2. `impl:` migration、repo、service、fetcher、handler。
3. `refactor:` 平台适配器接口整理。

## 6. 回滚策略

回滚 source migration，删除 source routes 和 fetcher 实现。
