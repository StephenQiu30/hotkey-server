# 事件聚类、生命周期与人工治理计划

## 计划目标

把匹配内容聚合为唯一全局 Event，并保证重跑、合并、拆分、人工锁和证据删除的事务正确性。

## 开工条件

- 对应 PRD 的 status 为 accepted，execution_status 为 ready
- frontmatter 中 depends_on 列出的 Plan 全部为 done
- main 已同步，工作区只包含当前任务相关文件

## 执行步骤

1. 固定跨语言事件标注集和 F1 计算。
2. 先写 event_key、重跑、生命周期、合并拆分与人工锁红灯测试。
3. 实现候选召回、event_fingerprint 和确定性成员阈值。
4. 实现 Event 创建、更新、迟到内容和代表内容。
5. 实现合并拆分事务及下游重算信号。
6. 发布 Event 查询与管理员治理 API。

## 验收清单

- 相同输入不创建重复 Event 或成员
- event_key 在标题和成员变化后保持稳定
- 无 LLM 时主链路仍可聚类
- 合并拆分失败不留下半迁移
- 人工锁不会被后续模型覆盖
- 决策可追溯到特征、版本、证据和决策者
