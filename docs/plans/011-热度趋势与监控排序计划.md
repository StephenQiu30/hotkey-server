# 热度、趋势与监控排序计划

## 计划目标

生成版本化 Event 指标快照，并在不同来源指标缺失或异常时保持确定性、可解释和可重算。

## 开工条件

- 当前 Plan 的 status 为 accepted、review_status 为 approved、execution_status 为 ready
- 对应 PRD 的 status 为 accepted，execution_status 为 ready
- frontmatter 中 depends_on 列出的 Plan 全部为 done
- main 已同步，工作区只包含当前任务相关文件

## 执行步骤

1. 固定来源能力、缺失指标、防刷和上升事件样本。
2. 先写确定性、封顶、快照唯一和证据删除红灯测试。
3. 实现来源指标归一化和缺失分量重分配。
4. 实现版本化热度、趋势和 Monitor 排序。
5. 实现证据删除、算法升级和指定窗口重算。
6. 更新 Event API、OpenAPI 和管理员算法配置。

## 验收清单

- 缺失指标不当作零
- 不直接比较不同来源原始互动数
- 单来源或单内容不能无界推高热度
- 相同输入、窗口和版本结果一致
- 历史快照不被算法升级改写
- 删除证据后热度与排序可幂等重算
