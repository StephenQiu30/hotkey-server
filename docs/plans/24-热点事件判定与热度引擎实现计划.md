# 热点事件判定与热度引擎实现计划（v1）

## 1. 目标

在不中断主链路的前提下，完成热度评分 v1 引擎、状态流转与 API 字段补齐。

## 2. Issue 映射

- #42 P0: 热点判定核心评分器（社媒热度+相关性）
- #43 P0: 热点判定可配置阈值与过滤规则
- #45 P0: 热点结果字段与响应 schema 补齐（含 hotness/risk 标识）

## 3. 任务拆解（含文件）

### T1 评分模块
- 新建：`apps/api/app/services/hotspot_scoring.py`
- 修改：`apps/api/app/core/settings.py`

输出：
- `compute_hotness`、`HOTNESS_*` 配置
- `hotness_breakdown` 与 `hotness_reason`

### T2 与检查链路耦合
- 修改：`apps/api/app/services/check_runner.py`
- 修改：`apps/api/app/services/search.py`

输出：
- 主流程打分和状态输出
- 列表排序主键升级为 `hotness_score`

### T3 字段与 API 映射
- 修改：`apps/api/app/schemas/hotspot.py`
- 修改：`apps/api/app/api/routes/hotspots.py`

输出：
- `hotness_*` 与 `source_risk_level` 在列表/详情可见

### T4 稳健回退
- 修改：`apps/api/app/models/hotspot.py`

输出：
- 异常默认值与降级路径

## 4. TDD 测试清单（先测后写）

- `test_compute_hotness_clamps_to_0_100`
- `test_hotness_high_relevance_marks_active`
- `test_hotness_low_reliability_becomes_filtered`
- `test_search_results_sorted_by_hotness_score`
- `test_run_hotspot_check_stores_hotness_fields`

## 5. 排序与验收

- 先补 `tests/test_mvp_services.py` 的单元与集成失败用例。
- 实现后执行回归，确保 API 兼容。
- 通过 `p0` 门禁：主链路 p95 回归、错误率不劣化、重复率不变更坏。

## 6. 风险与回滚

- 权重引起误过滤：支持 `HOTNESS_ACTIVE_THRESHOLD` 快速回退。
- 回退策略：默认值 + 缺失字段可空。
