# Scoring Topic Aggregation And Trend Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将已入库内容转化为可排序的热点内容流、主题聚合结果和趋势快照。

**Architecture:** 评分引擎先独立计算 `relevance_score`、`heat_score`、`freshness_score`、`author_influence_score`，再将结果喂给主题聚合器，最后由趋势模块按时间窗口沉淀快照。

**Tech Stack:** Go, PostgreSQL, SQL aggregations, table-driven tests

func ComputeHeatScore(in HeatInput) float64 {
    return float64(in.LikeCount)*1 +
        float64(in.ReplyCount)*3 +
        float64(in.RepostCount)*4 +
        float64(in.QuoteCount)*4 +
        float64(in.ViewCount)*0.02
}

## 风险与边界

1. 首版主题聚合使用规则和词元相似度，不引入 embedding 强依赖。
2. 趋势速度公式必须文档化，避免前后端解释不一致。
