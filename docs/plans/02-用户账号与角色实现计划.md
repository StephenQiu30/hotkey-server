# 02-用户账号与角色实现计划

## 1. 目标

提供邮箱注册登录、refresh token、当前用户信息和管理员鉴权边界。

## 2. 文件清单

- 创建：`migrations/000002_users.up.sql`
- 创建：`migrations/000002_users.down.sql`
- 创建：`internal/domain/user/`
- 创建：`internal/service/auth/`
- 创建：`internal/repository/postgres/userrepo/`
- 创建：`internal/transport/http/handlers/auth/`
- 修改：`internal/transport/http/router.go`
- 修改：`internal/config/config.go`

## 3. 任务拆解

1. migration 创建 `users`、`refresh_tokens`。
2. domain 定义用户角色、状态和邮箱值对象。
3. repository 实现用户创建、按邮箱查询、token hash 存取。
4. service 实现注册、登录、刷新、退出和当前用户查询。
5. HTTP handler 实现认证 API 和 admin middleware。
6. OpenAPI/contract 测试覆盖认证 API。

## 4. TDD 与验证

- 先写 service 单测：注册成功、邮箱重复、密码错误、普通用户访问 admin 返回 403。
- 再写 handler contract 测试。
- 再实现 repository 和 service。

## 5. 执行顺序

1. `test:` service 和 HTTP 失败测试。
2. `impl:` migration、domain、repository、service、handler。
3. `refactor:` 提取 token/hash 工具。

## 6. 回滚策略

回滚 migration，删除 auth handler 路由，并撤销新增 domain/service/repository。

## 7. 验收命令

```bash
gofmt -w cmd internal
go test ./...
python3 -m unittest discover -s tests
```
