# 17-内容标准化去重管线接通实现计划

## 1. 目标

让采集结果在进入 embedding 前完成统一标准化、过滤和去重，确保后续热点链路输入稳定。

## 2. 文件清单

- 修改：`internal/service/normalize/`
- 修改：`internal/service/filter/`
- 修改：`internal/service/dedup/`
- 修改：`internal/service/ingest/`

## 3. 任务拆解

1. 为重复内容、低质量内容和合法内容路径写失败测试。
2. 将 normalize/filter/dedup 串入采集后的统一处理流。
3. 记录保留、过滤和去重状态。
4. 确保仅新增有效内容进入后续 embedding 入队。
5. 运行验证并回写结果。

## 4. TDD 与验证

- 标准化和去重结果可追踪。
- 噪音和重复内容不会进入后续热点流程。
- 合法内容可以稳定进入 embedding 阶段。

## 5. 执行顺序

1. `test:` 重复、低质量、合法内容路径失败测试。
2. `impl:` 处理流串联与状态记录。
3. `refactor:` 统一 ingest 入口与下游入队逻辑。

## 6. 回滚策略

回滚 ingest 管线串联，保留各 service 独立实现；已处理内容状态保留。

## 7. 验收命令

```bash
go test ./internal/service/normalize/...
go test ./internal/service/filter/...
go test ./internal/service/dedup/...
gofmt -w cmd internal
go test ./...
python3 -m unittest discover -s tests
```

## 8. Symphony / Linear 要求

任务状态、标签和流转规则完全以本仓库 `WORKFLOW.md` 和本地 Symphony 实现为准。Plan 不定义额外状态、不发明额外标签、不覆盖 Symphony 的状态机。

Linear issue 只承载任务内容：PRD 路径、Plan 路径、任务范围、禁止范围、TDD 验收命令和回写要求。Symphony 负责监听 active states、创建 workspace、运行 Codex，并按 `WORKFLOW.md` prompt 驱动执行。
