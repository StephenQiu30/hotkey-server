# 多语言相关性匹配与反馈计划

## 计划目标

把 active Content 匹配到有限 Monitor 候选，保存可解释、版本化且可降级的相关性结果。

## 开工条件

- 当前 Plan 的 status 为 accepted、review_status 为 approved、execution_status 为 ready
- 对应 PRD 的 status 为 accepted，execution_status 为 ready
- frontmatter 中 depends_on 列出的 Plan 全部为 done
- main 已同步，工作区只包含当前任务相关文件

| 动作 | 路径 | 目的 |
|---|---|---|
| 创建 | internal/modules/ingestion/domain/match.go | MonitorMatch 与解释 |
| 创建 | internal/modules/ingestion/application/candidate_recall.go | 有限候选召回 |
| 创建 | internal/modules/ingestion/application/relevance.go | 混合评分 |

## 执行步骤

1. 先固定中英文、排除词、冲突实体和语义漂移标注集。
2. 写候选数量、Precision@20、解释和幂等红灯测试。
3. 实现词法与实体候选，再加入版本一致的向量召回。
4. 实现混合评分、高低阈值和 relevance_review 灰区路径。
5. 实现基于固定最近 Content 样本的无副作用相关性预览。
6. 实现反馈记录，不直接修改在线规则。
7. 发布匹配解释、预览、反馈和评测查询。

## 验收命令

| 阶段 | 命令 | 通过标准 |

| 质量 | go test ./internal/modules/ingestion/... -run RelevanceBenchmark -count=1 | Precision@20 不低于 80% |
| 降级 | HOTKEY_AI_DISABLED=1 go test ./internal/modules/ingestion/... -run RelevanceFallback | 词法与实体路径通过 |
| 全量 | make ci | 全部通过 |

## 验收清单

- 不对全部 Monitor 做笛卡尔比较
- 中英文内容可匹配同一意图
- 核心词权重大于 AI 扩展词
- 排除词和冲突实体能否决误报
- 每个结果保存分项、证据和版本
- 相同输入重跑不产生重复匹配
- 相关性预览不写 monitor_matches、Event 或下游 Job

## 提交边界

- test: 建立多语言相关性标注集与门禁
- impl: 实现候选召回和混合评分
- feat: 发布解释与反馈 API


## 风险与回滚

- 实现需要改变 Design 或 PRD 契约时立即停止，先更新正式文档并重新复核
- 下游 Plan 开工前，可整体回退本任务提交并同步恢复 Schema、OpenAPI 和测试
- 下游 Plan 已开工后，使用新的前向修复 Plan，不恢复旧双轨或兼容实现
