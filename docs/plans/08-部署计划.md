# 08 Deploy Plan

## 目标

提供基于本机 PostgreSQL 的自部署运行说明；Docker Compose 只作为 API/Web 可选启动方式。  
本地运行默认约束：
- 不使用 Python `venv` 作为前置条件（`python -m` 命令直接启动）。
- 不使用 Node 项目级包隔离环境（`npm` 直接执行开发命令）。
- PostgreSQL 与 Redis 使用本机 Homebrew 实例（`localhost`）。

## 范围

- API、Web 服务。
- PostgreSQL 使用用户本机已有实例，不由项目重新创建数据库容器。
- Redis 使用本机 Homebrew 实例；如需本地服务，可按默认 `redis://localhost:6379` 连接。
- 环境变量模板。
- 数据库初始化步骤。
- 数据库初始化与清理步骤。
- 可选 seed 默认来源和设置。

## 部署步骤

1. 复制 `.env.example` 为 `.env`。
2. 安装本机依赖（首次）：
   ```bash
   brew install postgresql redis
   brew services start postgresql
   brew services start redis
   ```
3. 在本机 PostgreSQL 中创建空数据库（不在仓库中启动 PostgreSQL 容器；如需 Docker 模式，数据库仍采用宿主机服务）。
4. 填写 PostgreSQL、模型 API、SMTP 配置。
5. 启动 API，或执行 `npm run db:init` 初始化表结构：
   ```bash
   python -m uvicorn server.app.main:app --reload --host 0.0.0.0 --port 8000
   ```
6. 可选执行 Docker Compose 启动 API/Web；容器仅接入宿主机 PostgreSQL/Redis（`host.docker.internal`）而不创建新数据库容器。
7. 打开控制台配置关键词。

## 验收标准

- 空 PostgreSQL 可以完成首次结构初始化。
- API 健康检查可访问。
- Web 控制台可访问。
- SMTP 未配置时系统仍可运行。
- 项目默认不启动新的 PostgreSQL 容器。

## 非目标

- 不提供云厂商一键部署。
- 不做旧数据导入。
