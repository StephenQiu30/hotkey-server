# 18 部署链路与健康可达计划

## 目标

建立 API/Web/PostgreSQL/Redis/Nginx 的最小可用部署流程和可达性验收顺序。

## 任务拆解

### B2-1 启动链路验证
- 执行 `docker compose -f docker-compose.prod.yml up -d`。
- 检查服务状态：
  - `api` 健康检查通过；
  - `web` 启动完成；
  - `postgres`、`redis` 健康通过。

### B2-2 可达性与主流程检查
- 验证 API 健康端点：`GET /api/health`。
- 验证前端入口：`/` 与 `/app` 可打开。
- 验证任务链路是否可触发（手动检查/调度入口）。

### B2-3 运维恢复动作
- 记录并验证：
  - restart / rebuild / rollback 的标准指令；
  - 关键服务不可用时的恢复时间。

### B2-4 交付复盘
- 记录截图/日志/时间轴。
- 更新运维异常清单：`docs/acceptance/B2-健康可达性问题清单.md`。

## 依赖

- 先前环境变量和 compose 文件可用。
- 推荐与 B1 的环境回放交叉执行。

## 验收标准

- compose 启动与终端健康都通过；
- 至少 1 次重启恢复可复现；
- 关键指令与结果存证完整。
