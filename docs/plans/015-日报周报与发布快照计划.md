# 日报、周报与发布快照计划

## 计划目标

按范围、周期和时区生成可追溯报告，发布后不因 Event 或算法变化而漂移。

## 开工条件

- 当前 Plan 的 status 为 accepted、review_status 为 approved、execution_status 为 ready
- 对应 PRD 的 status 为 accepted，execution_status 为 ready
- frontmatter 中 depends_on 列出的 Plan 全部为 done
- main 已同步，工作区只包含当前任务相关文件

## 执行步骤

1. 先写跨时区周期、幂等键、发布不漂移和证据红灯测试。
2. 同步 Report、Item 与冻结版本 Schema。
3. 实现周期计算、事件选择和不可变 report_item。
4. 实现 report_draft 与无 LLM 结构化降级。
5. 复用 Knowledge 原子写入发布 Vault。
6. 接入 build_report Job 与安全管理 API。

## 验收清单

- 同一范围和周期只产生一个有效报告版本
- 周期按订阅时区计算，数据库保存 UTC
- 发布后 Event 更新不改变报告内容
- 核心结论可定位到冻结证据
- LLM 不可用时仍可发布结构化报告
- 日报周报路径与 Frontmatter 稳定

## 提交边界

- test: 定义报告周期与快照门禁
- impl: 实现报告选择、构建和持久化
- feat: 接入 Vault 发布与 build_report Job


## 风险与回滚

- 实现需要改变 Design 或 PRD 契约时立即停止，先更新正式文档并重新复核
- 下游 Plan 开工前，可整体回退本任务提交并同步恢复 Schema、OpenAPI 和测试
- 下游 Plan 已开工后，使用新的前向修复 Plan，不恢复旧双轨或兼容实现
