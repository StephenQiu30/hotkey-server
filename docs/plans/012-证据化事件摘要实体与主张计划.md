# 证据化事件摘要、实体与主张计划

## 计划目标

为 Event 生成结构化双语摘要、时间线、实体与主张，并拒绝无法定位到 active Content 的结论。

## 开工条件

- 当前 Plan 的 status 为 accepted、review_status 为 approved、execution_status 为 ready
- 对应 PRD 的 status 为 accepted，execution_status 为 ready
- frontmatter 中 depends_on 列出的 Plan 全部为 done
- main 已同步，工作区只包含当前任务相关文件

## 执行步骤

1. 固定有效引用、非法引用、证据撤回和人工锁样本。
2. 先写 JSON Schema、引用定位、复用失效和降级红灯测试。
3. 实现实体、别名、关系、主张和证据领域模型。
4. 实现 event_summary 与 entity_claim_extraction。
5. 校验所有输出引用，非法结果不得进入正式版本。
6. 实现人工修正、重新生成和无 LLM 降级摘要。

## 验收清单

- 核心结论均定位到 active Content
- 非法引用无法发布
- 证据删除后旧结果不复用
- 人工锁不被模型覆盖
- 无 LLM 时 Event、证据和热度 API 仍可用
- 历史 ai_run 不被人工修正覆盖

## 提交边界

- test: 定义事件智能引用与幻觉门禁
- impl: 实现实体、主张和证据校验

## 风险与回滚

- 实现需要改变 Design 或 PRD 契约时立即停止，先更新正式文档并重新复核
- 下游 Plan 开工前，可整体回退本任务提交并同步恢复 Schema、OpenAPI 和测试
- 下游 Plan 已开工后，使用新的前向修复 Plan，不恢复旧双轨或兼容实现
