# 证据化事件摘要、实体与主张计划

## 计划目标

为 Event 生成结构化双语摘要、时间线、实体与主张，并拒绝无法定位到 active Content 的结论。

## 开工条件

- 当前 Plan 的 status 为 accepted、review_status 为 approved、execution_status 为 ready
- 对应 PRD 的 status 为 accepted，execution_status 为 ready
- frontmatter 中 depends_on 列出的 Plan 全部为 done
- main 已同步，工作区只包含当前任务相关文件

## 任务边界

本计划只交付事件智能结果及其证据闭环；不实现 River 编排、Vault 投影、报告或投递。既有 `entities`、`event_entities`、`event_claims`、`claim_evidences`、AI Run、Schema 与人工 Claim API 是可复用基础，不重新创建平行模型。

## Task 1：冻结证据输入和结构化结果契约

**输入/输出：** `EventID + active evidence set + version` → 受限 `event_summary` / `entity_claim_extraction` JSON；每个引用必须使用当次输入的 content ID、locator 与短摘录。

- [x] **RED：** 为未知引用、失效 Content、越界 locator、未知 JSON 字段和无证据结论添加失败测试。
- [x] **GREEN：** 注册静态 Schema，验证实体/主张/摘要值对象并生成稳定 evidence/input hash。
- [x] **重构：** 让 Schema、领域校验和 HTTP DTO 共用受限字段，不让 Provider 原始响应穿透。
- [x] **回归：** `go test ./internal/modules/event/domain ./internal/modules/event/application ./internal/modules/intelligence/application -run 'Evidence|Summary|Claim|Entity|Schema' -count=1`。
- [x] **提交：** `test: define event intelligence evidence contracts`。

**输入/输出：** 已验证证据快照 → 已验证的 `ai_runs.structured_result` 或明确的降级摘要；相同 reuse key 复用成功结果，失效证据不得复用。

- [x] **RED：** 为复用、无可用 Provider、一次 repair、结果引用校验和证据变更失效添加应用测试。
- [x] **GREEN：** 通过 Intelligence Application 调用既有 RunService，保存仅含安全结构化结果的运行事实；Provider 不可用时返回代表 Content 的降级摘要。
- [x] **重构：** Event 仅依赖 Intelligence Application 接口，不直接访问 AI 表或 Provider。
