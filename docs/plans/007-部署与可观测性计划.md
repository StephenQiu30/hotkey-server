# Deployment And Observability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 API、Worker、PostgreSQL 和 Redis 能本地一键运行，并具备基础健康检查、结构化日志、关键指标和验收脚本。

**Architecture:** 使用 Docker Compose 编排四个核心服务，并通过统一验证脚本和验收文档提供可重复的本地联调路径。观测能力先做日志、健康检查和关键计数器，不先上复杂平台。

**Tech Stack:** Docker Compose, Go logging, health endpoints, shell scripts, acceptance docs

## 验收门禁

1. 本地可以一键起服务。
2. 健康检查和结构化日志可见。
3. 验证脚本覆盖后端测试、schema 和 compose 配置。
4. 验收文档能指导完整联调。

## 风险与边界

1. 首版不要求接入正式监控平台，日志和脚本足够支撑开发验证。
2. 生产级部署脚本可在 MVP 稳定后再单独扩展。
