# 消息来源真伪验证与证据链实现计划（v1）

## 1. 目标

把来源真伪评估结果接入检测链路，在不阻断的前提下支持证据可视化、降权与复盘。

## 2. Issue 映射

- #46 P1: 消息来源真伪证据采集器
- #47 P1: 来源真伪评分器（证据优先、降权策略）
- #48 P1: 证据链落库与归因字段设计
- #49 P1: 低置信来源展示策略与风控标记

## 3. 任务拆解（含文件）

### T1 证据采集器
- 新建：`apps/api/app/services/source_trust.py`
- 修改：`apps/api/app/services/ingestion.py`

### T2 真伪评分
- 修改：`apps/api/app/services/source_trust.py`
- 修改：`apps/api/app/services/ai_analysis.py`

### T3 持久化与接口
- 修改：`apps/api/app/models/ai_analysis.py`
- 修改：`apps/api/app/schemas/ai_analysis.py`
- 修改：`apps/api/app/api/routes/hotspots.py`

### T4 降权与展示
- 修改：`apps/api/app/services/check_runner.py`
- 修改：`apps/api/app/services/notification_service.py`（若存在）

## 4. TDD 测试清单（先测后写）

- `test_truth_score_allows_degraded_mode`
- `test_source_domain_risk_labeling`
- `test_cross_source_count_in_evidence`
- `test_low_trust_penalty_affects_hotness`
- `test_low_trust_event_filtered_for_notification`
- `test_evidence_bundle_visible_in_search_and_list`

## 5. 验收

- 给出证据链可追溯字段。
- 低置信必须保留风险标签并降权。
- 不得阻断抓取主链路与邮件流程可控。

## 6. 风险与回滚

- 证据字段过重导致解析成本增加：采用 JSON 版本化与字段白名单。
- 回退：`LOW_TRUST_PENALTY` 默认 0（无降权）。
