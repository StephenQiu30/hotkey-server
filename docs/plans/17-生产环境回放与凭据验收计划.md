# 17 生产环境回放与凭据验收计划

## 目标

把企业级 P0 B1 的真实环境回放转为一次可复用、可审计的执行计划。

## 任务拆解

### B1-1 真实环境准备
- 补齐并校验 `.env.example/.env` 中的凭据项：
  - `DATABASE_URL`
  - `X_API_BEARER_TOKEN`
  - `BING_SEARCH_API_KEY`
  - `OPENAI_API_KEY` `OPENAI_BASE_URL` `OPENAI_MODEL`
  - `SMTP_*`
- 验证 PostgreSQL 空库初始化成功（`sql/001_init_schema.sql`）。
- 输出：`env-check.md`。

### B1-2 实时回放执行
- 启动 PostgreSQL + api + web。
- 创建/启用至少 2 个关键词与 2 个来源。
- 触发一次检查任务。
- 检查：`hotspots`、`check_runs`、`notifications`、`reports` 数据一致。
- 输出：`runbook/B1-run-logs.md`（命令、时间、结果）。

### B1-3 外部服务验证
- 分别验证真实 X/Twitter、Bing、模型、SMTP 至少各一次成功路径。
- 若某项不可达，记录：
  - 失败原因；
  - 已执行重试动作；
  - 是否影响主闭环。

### B1-4 证据归档
- 按 `docs/验收差距清单.md` 格式，补齐“真实凭据验收”各项。
- 形成可复验工单清单：`docs/acceptance/B1-回放证据包.md`。

## 依赖

- 上游：A2/A4/A8 逻辑完成。
- 并行：可与 B2 启动和健康性验证并行。

## 验收标准

- 每个外部能力至少各有一次成功调用记录（或明确不可达证据）；
- 回放流程结束时存在一条检查任务成功记录；
- 报告生成、邮件发送路径有可追溯日志。

## 风险

- 凭据不可用导致回放无法全通，属于验收阻塞项；
- 外部速率限制导致采集样本不足。
