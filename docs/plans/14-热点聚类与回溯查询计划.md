# 14-热点聚类与回溯查询计划

## 目标

补齐企业级 A4：让聚类可追溯、可复盘、可查询版本历史。

## 范围与不变项

- 范围：
  - 聚类版本元数据补齐（`cluster_version`）；
  - 新增聚类查询接口；
  - 输出列表支持按时间线排序与分页。
- 不变项：
  - `URL + source` 去重逻辑保持不变；
  - 现有热点列表排序与筛选 API 不变。

## 任务拆解

- **P0-A4-1：聚类版本元数据写入**
  - 在 `server/app/services/check_runner.py`：
    - 引入 `_next_cluster_version`；
    - `candidate.raw_payload` 补齐 `cluster_id`、`cluster_version`、`clustered_at`（可选）。

- **P0-A4-2：聚类历史 API**
  - 在 `server/app/api/routes/hotspots.py` 增加：
    - `GET /api/hotspots/cluster/{cluster_id}`
    - `GET /api/hotspots/{hotspot_id}/cluster-history`
  - 返回结构包含：
    - `cluster_id`、`cluster_size`、`items[]`（含关键时间和评分字段）。

- **P0-A4-3：测试与回归**
  - 新增 `tests/test_mvp_services.py`：
    - 同一 cluster id 下能返回历史；
    - 无历史时返回空结果。
  - 覆盖 `cluster_version` 单调递增或默认值逻辑。

## 依赖与顺序

- 先补 `check_runner` 元数据，再开接口；
- 先补 `test:`，再 `impl:`。

## 验收

- 单测通过；
- 聚类相关 API 响应包含 `cluster_id` + `cluster_version`；
- `docs/engineering/验收标准.md` 的 A4 更新为“已完成/待复核”。
