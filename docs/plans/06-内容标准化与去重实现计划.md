# 06-内容标准化与去重实现计划

## 1. 目标

把不同来源内容统一为 `source_items`，按 URL/hash/title 去重，并为新增内容触发 embedding。

## 2. 文件清单

- 创建：`migrations/000006_source_items.up.sql`
- 创建：`internal/domain/content/`
- 创建：`internal/service/ingest/`
- 创建：`internal/repository/postgres/contentrepo/`
- 创建：`internal/worker/handlers/collect/`

## 3. 任务拆解

1. 创建 `source_items` 表和唯一约束。
2. 实现 canonical URL 和 content hash。
3. 实现 ingest service 标准化和去重。
4. collect worker 调用 source service 和 ingest service。
5. 新增 item 入队 `generate_embedding`。

## 4. TDD 与验证

- 同一 canonical URL 重复入库只保留一条。
- 同 hash 不同 URL 标记重复。
- 新 item 创建 embedding job。

## 5. 执行顺序

1. `test:` content repo 和 ingest service 失败测试。
2. `impl:` migration、domain、repo、service、worker handler。
3. `refactor:` 去重策略提取。

## 6. 回滚策略

回滚 `source_items` migration，停用 collect worker handler。
