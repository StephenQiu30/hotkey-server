# 25-n8n目录凭证与导入说明实现计划

## 1. 目标

新增仓库级 `n8n/` 目录，用于保存可导入 workflow 模板和导入说明，同时明确凭证不入库。

## 2. 文件清单

- `n8n/README.md`
- `n8n/workflows/*.json`
- `.env.example`

## 3. 任务拆解

- 创建 `n8n/README.md`，说明本地 n8n 地址、导入步骤、Credentials 名称和安全边界。
- 创建 `n8n/workflows/`，只保存模板 JSON。
- workflow 中使用占位变量，不保存真实 key、SMTP 密码或 OAuth token。
- README 中列出与 hotkey-server 对接的 internal API 和 header。

## 4. TDD 与验证

- secret scan 不出现真实密钥格式。
- workflow JSON 可被 n8n 导入。
- README 中的变量与 `.env.example` 保持一致。

## 5. 执行顺序

1. 先创建 README 和目录。
2. 再添加 workflow 模板。
3. 最后执行 secret scan 和导入检查。

## 6. 回滚策略

- 删除 `n8n/` 模板目录不影响后端运行。
- 凭证只在 n8n 实例或部署环境中维护。

## 7. 验收命令

```bash
rg -n "password|secret|token|apikey" n8n .env.example
```
