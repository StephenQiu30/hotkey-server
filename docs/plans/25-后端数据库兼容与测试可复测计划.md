# 25 后端数据库兼容与测试可复测计划

## 目标

将后端数据库连接与模型层在生产、测试、开发的行为统一为可复现路径：

- 生产：继续使用 PostgreSQL + `sql/001_init_schema.sql`。
- pytest / 本地无 PostgreSQL：自动使用 SQLite fallback，保证回归测试可跑。

- 修改：

  - 重构 `apps/api/app/db/init_schema.py`
- 实现点：
  - 新增 `is_pytest_run()` 判定；

- 测试：

## 依赖关系与顺序

1. 先实现连接策略；
2. 再完成初始化策略联通；
3. 最后修复 JSON 查询/字段兼容。
