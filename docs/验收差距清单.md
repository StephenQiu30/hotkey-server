## 真实凭据验收

- X/Twitter：填写 `X_API_BEARER_TOKEN` 后，验证官方 X API v2 Recent Search 来源能返回候选；未填写时只跳过该来源。
- Bing：填写 `BING_SEARCH_API_KEY` 后，验证 Bing Search 来源能返回候选；未填写时只跳过该来源。
- 模型服务：填写 `OPENAI_API_KEY`、`OPENAI_BASE_URL`、`OPENAI_MODEL` 后，验证查询扩展和热点分析使用真实模型；未填写时使用本地降级逻辑。
- SMTP：填写 `SMTP_HOST`、`SMTP_USERNAME`、`SMTP_PASSWORD`、`SMTP_FROM_EMAIL`、`SMTP_TO_EMAIL` 后，验证事件邮件和报告邮件发送；未填写时记录 `skipped`。
- PostgreSQL：使用真实本机数据库连接串 `DATABASE_URL` 初始化空库并完成一次端到端检查。

## 人工端到端验收

- 创建、启停、删除关键词。
- 启用至少两个来源并手动触发热点检查。
- 验证同一来源和 URL 不重复入库。
- 验证 `active` / `filtered` 阈值过滤。
- 验证 `filtered` 热点不发送事件邮件、不进入日报/周报。
- 生成 daily 和 weekly 报告，并确认报告内容来自本地 Markdown 模板。

## 明确不进入 P0

- 多用户、租户隔离、真实认证、真实计费。
- 实时推送、复杂通知策略、复杂工作流。
- 向量库、Redis、Celery 或独立复杂队列平台。
- AI 生成整篇报告正文作为必需能力。
